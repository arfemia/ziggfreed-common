A reusable cast / ability RUNTIME layer (`cast/`), lifted config-free from a consumer: the on-hit builder dispatch, the per-caster armed-state store, ray + block targeting, and the whole per-world tick partition (frame gate, world-keyed queues, evictor fan-out, and the abstract drain system). So a mod that fires abilities reuses ONE battle-tested set of runtime seams instead of re-deriving them. All additive over 1.3.0; nothing here is breaking, and the layer ships EMPTY of content - every built-in on-hit type and every concrete drain system is consumer policy.

- **New: `cast/OnHitRegistry` - a per-consumer on-hit builder registry.** News up your own instance so two mods sharing this jar never see each other's type registrations. `register(type, builder)` keys case-insensitively (last write wins); `fromParams(shape, sourceRef, sourcePlayerId)` converts an `onHit` JSON shape - a single object OR a list chained in declaration order (unresolved entries skipped) - into one runnable callback. An unknown type or a builder that throws degrades to a shared no-op sentinel with a guarded log. Ships with ZERO registered types.
- **New: `cast/CastParams` - lenient param coercion.** `numberOr` / `stringOr` / `boolOr` read one key from a params bag and fall back on an absent or mistyped value, never throwing. Pure, no engine.
- **New: `cast/ArmedStateStore` - a per-consumer per-caster combat-state store.** Next-hit damage multipliers / flat damage / on-hit chains keyed by `(casterId, slot)`, plus a parallel invulnerability window. Consuming synthesizes one composite across all live slots (multipliers multiply, flats sum, on-hit callbacks chain, id lists union-deduplicate), with expiry-on-read pruning. The caller supplies the key and the store NEVER derives it - the consumer's caster-context stays the single key-derivation authority.
- **New: `cast/RaycastTargeting` + `cast/BlockRaystep` - stateless targeting utilities.** Entity-raycast picking (nearest / all-piercing, caster excluded) and a look-ray block walker (clear distance with optional wall pullback / exact hit point); an unloaded block is treated as clear. All radii / step / pullback values are caller-supplied.
- **New: the per-world tick partition.** `cast/WorldFrameGate` (a once-per-millisecond CAS gate, one per world), `cast/WorldEvictors` (the JVM-global world-resolution + eviction fan-out), `cast/WorldKeyedQueues` (a generic per-world queue partition that self-registers its eviction, with a one-time warning when entries are added but no drain system has attached), and `cast/AbstractWorldFrameSystem` (an abstract ticking-system base that gates per world and calls one `drainFrame` hook). The abstract system is NEVER registered by common - the ECS system registry is class-keyed, so each consumer registers its own concrete drain subclass.
- **New: `cast/ModelParticleService` - guarded particle spawns.** Position-only and rotation-aware wrappers over the engine particle util; a null asset is a no-op, any error is caught and reported at a low log level, and the call returns a success boolean. World-thread.
