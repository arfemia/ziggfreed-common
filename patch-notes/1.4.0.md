A reusable cast / ability RUNTIME layer (`cast/` + `cast.step/`), lifted config-free from a consumer: the generic step-dispatch kernel, the on-hit builder dispatch, the per-hit resolver, the per-caster armed-state store, a guarded observer registry, ray + block targeting, and the whole per-world tick partition (frame gate, world-keyed queues, evictor fan-out, and the abstract drain system). So a mod that fires abilities reuses ONE battle-tested set of runtime seams instead of re-deriving them. All additive over 1.3.0; the layer ships EMPTY of content and of any step vocabulary - every built-in step / on-hit type and every concrete drain system is consumer policy, and the kernel is generic over the consumer's own context / step / result types. The `cast/` surface is additive-only once frozen. (This 1.4.0 is UNRELEASED / held - a tag or GitHub push is not a release.)

- **New: `cast.step` - a generic ordered step-dispatch kernel.** `StepHandler<C,S,R>` (a handler SAM), `StepRegistry<K,C,S,R>` (a per-consumer, ships-empty, last-write-wins register/get map with an enum-backed subclass path), `StepSemantics<C,S,K,R>` (the tiny consumer-supplied contract adapting a consumer's existing spec / step / result methods with no `implements` on them), and `CastKernel<C,S,K,R>.run(ctx)` (walk in order, dispatch by key, short-circuit on the first non-success result, empty chain succeeds, missing handler deferred to the consumer's hook). Common defines no step type of its own; the walk is byte-parity with a hand-written dispatch loop.
- **New: `cast/OnHitRegistry` - a per-consumer on-hit builder registry.** News up your own instance so two mods sharing this jar never see each other's type registrations. `register(type, builder)` keys case-insensitively (last write wins); `fromParams(shape, sourceRef, sourcePlayerId)` converts an `onHit` JSON shape - a single object OR a list chained in declaration order (unresolved entries skipped) - into one runnable callback. A parallel `registerAction` / `actionFromParams` path (separate table) produces the richer `HitContext`-carrying callback instead, for a hook that needs a command-buffer accessor / damage amount / hit position. An unknown type or a builder that throws degrades to a shared no-op sentinel with a guarded log. Ships with ZERO registered types.
- **New: `cast/HitContext` + `cast/HitAction` + `cast/HitResolver` - an evolvable per-hit application seam.** `HitContext` is a field-additive hit carrier whose accessor is a `ComponentAccessor` (so a live store AND a command buffer both fit - the deferred-mutation seam), replacing a raw two-argument on-hit callback that could carry neither a command buffer nor a hit position. `HitAction` is the callback over it (with a no-op + chain). `HitResolver.applyHit` runs one resolved hit: the damage dispatch under an opaque origin decorator, then the on-hit hook (a `HitAction` or a bridged two-argument callback), then an opaque per-hit sound sink and an optional particle; it reports whether the dispatch threw. Every per-caller knob rides an immutable request builder, and the caller's own sink builds the engine damage, so nothing consumer-specific enters the library.
- **New: `cast/ObserverRegistry` - a per-consumer guarded observer registry.** `register` + `fire` over a copy-on-write list with a per-listener guard, so one misbehaving listener neither throws out of `fire` nor blocks the others.
- **New: `cast/CastParams` - lenient param coercion.** `numberOr` / `stringOr` / `boolOr` read one key from a params bag and fall back on an absent or mistyped value, never throwing. Pure, no engine.
- **New: `cast/ArmedStateStore` - a per-consumer per-caster combat-state store.** Next-hit damage multipliers / flat damage / on-hit chains keyed by `(casterId, slot)`, plus a parallel invulnerability window. Consuming synthesizes one composite across all live slots (multipliers multiply, flats sum, on-hit callbacks chain), with expiry-on-read pruning. Each arm carries an OPAQUE attribution payload the consumer attaches and reads back off the composite (exposed per live slot in encounter order for the consumer to fold), so no consumer-domain vocabulary lives in the library; the payload-free convenience overload is retained and the state views are immutable. The caller supplies the key and the store NEVER derives it - the consumer's caster-context stays the single key-derivation authority.
- **New: `cast/RaycastTargeting` + `cast/BlockRaystep` - stateless targeting utilities.** Entity-raycast picking (nearest / all-piercing, caster excluded) and a look-ray block walker (clear distance with optional wall pullback / exact hit point); an unloaded block is treated as clear. All radii / step / pullback values are caller-supplied.
- **New: the per-world tick partition.** `cast/WorldFrameGate` (a once-per-millisecond CAS gate, one per world), `cast/WorldEvictors` (the JVM-global world-resolution + eviction fan-out), `cast/WorldKeyedQueues` (a generic per-world queue partition that self-registers its eviction, with a one-time warning when entries are added but no drain system has attached), and `cast/AbstractWorldFrameSystem` (an abstract ticking-system base that gates per world and calls one `drainFrame` hook). The abstract system is NEVER registered by common - the ECS system registry is class-keyed, so each consumer registers its own concrete drain subclass.
- **New: `cast/ModelParticleService` - guarded particle spawns.** Position-only and rotation-aware wrappers over the engine particle util; a null asset is a no-op, any error is caught and reported at a low log level, and the call returns a success boolean. World-thread.
