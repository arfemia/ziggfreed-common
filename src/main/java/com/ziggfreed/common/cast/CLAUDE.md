# cast/ - reusable cast / ability runtime primitives

Router for `com.ziggfreed.common.cast` (+ `cast.step`). The mod-agnostic runtime seams a consumer's cast / ability system needs: the generic step-dispatch KERNEL, an on-hit builder registry, a per-hit resolver, a per-caster armed-state store, a guarded observer registry, ray + block targeting, and the per-world tick partition (frame gate, world-keyed queues, evictor fan-out, and the abstract drain system). Lifted config-free from a consumer; every engine-touching call is try-guarded to a no-op / `false`, and world-thread work is asserted in javadoc, not enforced. No content, no ids, no step vocabulary baked in - the consumer supplies all types and registers all policy. The layer is ADDITIVE-ONLY once frozen (new methods / overloads / `HitContext` fields OK; renames / removals / callback-widening forbidden).

## Step-dispatch kernel (`cast.step`)

The generic ordered-step-walk engine, parameterized over a consumer's OWN context `C`, step `S`, key `K`, and result `R` - common defines no step type of its own.

- **[`StepHandler<C,S,R>`](step/StepHandler.java)** - the SAM a consumer's concrete step handlers implement: `R execute(C ctx, S step)`.
- **[`StepRegistry<K,C,S,R>`](step/StepRegistry.java)** - a PER-CONSUMER `register(key, handler)` / `@Nullable get(key)` map (ships EMPTY, last-write-wins, null-key-safe). No-arg ctor = `ConcurrentHashMap` backing; a protected `StepRegistry(Map)` ctor lets an enum-keyed subclass supply an `EnumMap`.
- **[`StepSemantics<C,S,K,R>`](step/StepSemantics.java)** - the tiny consumer-supplied contract (`stepsOf` / `keyOf` / `isSuccess` / `successResult` / `onMissingHandler`) that adapts a consumer's existing spec / step / result methods to the walk WITHOUT any `implements` on those types. The missing-handler hook stays consumer-side so a consumer keeps its own logging (its own ids) + its own error result.
- **[`CastKernel<C,S,K,R>`](step/CastKernel.java)** - `new CastKernel(registry, semantics).run(ctx)`: walk steps in order, look each handler up by key, short-circuit on the first non-success result, empty chain succeeds, missing handler returns the consumer's `onMissingHandler` result. Byte-parity with a hand-written dispatch loop; does NOT catch a handler throw.

- **[`OnHitRegistry`](OnHitRegistry.java)** - a PER-CONSUMER INSTANCE registry (news up your own; two mods sharing this jar never see each other's types). TWO parallel callback shapes in SEPARATE tables: the original `register(type, OnHitBuilder)` + `fromParams(Object|Map, sourceRef, sourcePlayerId)` -> `BiConsumer<Store, Ref>`, AND `registerAction(type, HitActionBuilder)` + `actionFromParams(...)` -> `HitAction` (a `HitContext` carrier that can hold a `CommandBuffer` accessor / damage amount / hit position, for a callback that needs more than store + target). Both case-insensitive, last-write-wins; both chain a list via `andThen` (`NO_OP` / `NO_OP_ACTION` entries skipped); unknown type / thrown builder -> the matching no-op + guarded FINE log. A type registered on one table is invisible to the other. Ships EMPTY.
- **[`HitContext`](HitContext.java)** - the immutable, FIELD-ADDITIVE hit carrier: `accessor` (a `ComponentAccessor<EntityStore>`, so a live `Store` AND a `CommandBuffer` both fit - the deferred-mutation seam), `target`, `source`, `sourcePlayerId`, `damageAmount`, `position`, `cause`. Builder + getters; a new field never breaks a `HitAction` implementer (unlike widening a JDK `BiConsumer` generic). The evolvable successor to a raw `BiConsumer<Store, Ref>` on-hit hook.
- **[`HitAction`](HitAction.java)** - `void onHit(HitContext)` + `NO_OP` + a `andThen` chain (drops a null / `NO_OP` operand).
- **[`HitResolver`](HitResolver.java)** - a stateless per-already-resolved-target hit routine: `applyHit(store, HitRequest, target, hitPos)` runs the damage dispatch under an opaque `UnaryOperator<Runnable>` origin decorator (wraps EXACTLY the dispatch), then the onHit hook (a `HitAction` OR a bridged `BiConsumer<Store, Ref>`, no context alloc on the bridge path), then an opaque per-hit sound sink (`BiConsumer<Store, Vector3d>`) and a per-hit particle via `ModelParticleService`; returns whether the dispatch threw (WARN/FINE per request). The `DamageSink` builds the engine `Damage` itself (from the request's source/cause/amount), so common constructs no `Damage` - keeps it engine-light + unit-testable. All the per-caller policy knobs live on the `HitRequest` builder.
- **[`ObserverRegistry<E>`](ObserverRegistry.java)** - a PER-CONSUMER `register(Consumer<E>)` / `fire(E)` over a `CopyOnWriteArrayList` with a PER-LISTENER guard (one throwing listener neither throws out of `fire` nor blocks the rest). Optional label ctor for the guarded warn.
- **[`CastParams`](CastParams.java)** - static param-coercion helpers for a `Map<String,Object>` params bag: `numberOr` (-> `Number`), `stringOr` (-> nullable `String`), `boolOr`. Lenient: a mistyped value falls back, never throws. Pure, no engine.
- **[`ArmedStateStore`](ArmedStateStore.java)** - a PER-CONSUMER INSTANCE store of per-caster transient armed state (next-hit multipliers / flat damage / on-hit chains keyed `(casterId, slot)`) + a parallel invulnerability window. `armNextHit`/`armMultiplier`/`peekArmedMultiplier`/`consumeArmed` (composite: multipliers multiply, flats sum, on-hit consumers chain, each live slot's OPAQUE `attribution` payload exposed in encounter order via `ArmedState.attributions` for the consumer to fold) + `armInvulnerability`/`isInvulnerable`/`peekInvulnerability` + `clearPlayer`. Each arm / invuln call carries a `@Nullable Object attribution` (retained payload-free `armMultiplier(UUID,double,long)` overload); `ArmedState`/`InvulnerabilityState` are final-field + ctor. No consumer-domain vocabulary lives here. Expiry-on-read pruning. **The caller supplies the UUID key and this store NEVER derives it - the consumer's caster-context stays the single key-derivation authority.** Pure state, no engine calls (unit-testable with null refs).
- **[`RaycastTargeting`](RaycastTargeting.java)** - stateless static entity-raycast picking: `pickClosest` (nearest hit) / `pickPiercing` (all hits closest-first, capped) over `Selector` + `CollisionMath.intersectRayAABB`, caster excluded by index; `inflateRadius` widens only the broad-phase search. `Hit` carries the ref + ray-t + pick-time position. World-thread.
- **[`BlockRaystep`](BlockRaystep.java)** - stateless static look-ray block walker: `clearDistance` (distance to first Solid block, optional wall pullback) / `hitPosition` (exact hit point). All step / distance / pullback values are caller-supplied; an unloaded / off-map block is treated as clear (`catch (Throwable ignored)` keeps stepping). World-thread.
- **[`WorldFrameGate`](WorldFrameGate.java)** - a once-per-world-frame CAS gate on the millisecond clock: `beginFrame()` is `true` only for the first call per millisecond. Hold ONE per world. Thread-safe; no world-thread requirement.
- **[`WorldEvictors`](WorldEvictors.java)** - STATIC (JVM-global, so fan-out reaches every consumer's partitions in one process): `worldOf(Store|Ref)` world resolution + `registerEvictor(Consumer<World>)` + `onWorldRemoved(World)` guarded fan-out. A consumer's `RemoveWorldEvent` listener calls `onWorldRemoved`.
- **[`WorldKeyedQueues`](WorldKeyedQueues.java)** - a generic per-world queue partition `WorldKeyedQueues<E>`: `queueFor(world)` (computeIfAbsent, add path) / `peek(world)` (nullable get, drain path) / `isEmpty` / `values()` / `totalSize` / `evict`. Self-registers a `WorldEvictors` evictor in its constructor. A drain system MUST call `markDrainerAttached()`; a one-time WARNING fires if entries are added before any drainer attaches. Each per-world cast ticker swaps its private map for one instance with no tick-body change.
- **[`AbstractWorldFrameSystem`](AbstractWorldFrameSystem.java)** - abstract `EntityTickingSystem<EntityStore>` base (player-query gate). Resolves the world from the store, gates via an internal per-world `WorldFrameGate` (self-evicting), and on winning calls the protected abstract `drainFrame(world, store, commandBuffer)`. **NEVER registered by common - the ECS system registry is class-keyed, so each consumer registers its OWN concrete subclass.**
- **[`ModelParticleService`](ModelParticleService.java)** - static `ParticleUtil` wrappers: `spawnAt(store, asset, pos)` (position-only) + `spawnDirectional(store, asset, pos, rotation)` (collects nearby players to reach the rotation-aware overload). Null asset -> `false` no-op; `catch (Throwable)` -> `false` + guarded FINE log; `true` on success. World-thread.
