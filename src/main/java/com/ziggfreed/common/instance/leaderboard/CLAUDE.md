# instance/leaderboard/ - the generic bucketed leaderboard primitive

Router for `instance/leaderboard/`. The mod-root `CLAUDE.md` instance-experience section is the wider context. A mod-agnostic, JSON-persisted, UUID-keyed leaderboard + its page; the consumer supplies only policy (the board, the tab axes, the stat columns, the chrome). Kweebec's wiring is its `experience/` package.

- **[`Leaderboard`](Leaderboard.java)** - the store: `record(bucket, uuid, name, score, time, win[, statDeltas])` mutates the in-memory `ConcurrentHashMap` immediately (world-thread-safe) and schedules a DEBOUNCED off-thread atomic flush (`FileUtil.writeStringAtomic`, `.bak` fallback). Per record it keeps `bestScore` (max), `bestTimeSeconds` (min WINNING), `plays`, `totalPoints` (cumulative sum), and merges the consumer-keyed `stats` map. `forBucket(key)` = one bucket; `forBuckets(keys)` = a GLOBAL per-player aggregate across buckets (the lifetime/stats view). Bucket key is a free string - the consumer's strategy (Kweebec: `"<difficulty>_<partySize>"`; one board PER game-mode, `"leaderboard-<mode>"`).
- **[`LeaderboardEntry`](LeaderboardEntry.java)** - the persisted per-player shape (public Gson fields). `stat(key)` defaults absent counters to 0; `stats`/`totalPoints` are null/0 on legacy entries.
- **[`SortMode`](SortMode.java)** - the three rank metrics (BEST_SCORE / TOTAL_POINTS / BEST_TIME) each carrying a `comparator()` (best-first; no-win times sort last) + a `metric()` for the footer.
- **[`StatColumnDef`](StatColumnDef.java)** - one Stats-view column (stat key + `Message` label + `ColumnFormat`). The stats-row `.ui` ships `MAX_STAT_COLUMNS` (4) generic cells; extra columns are dropped.
- **[`LeaderboardPage`](LeaderboardPage.java)** - the page: an optional PRIMARY tab row (`#GroupBar`) over the SECONDARY tab row (`#TabBar`), a sort toggle + Rankings/Stats view toggle, and the ranked/stats list. STATELESS across events - every binding round-trips the full `(group, bucket, sort, view)` state via `LeaderboardEventData`, and `handleDataEvent` reopens the page with the next state (every exit path sends a response). Empty `primaryTabs`/`statColumns` degrade to the original single-axis, score-only board. The `(playerRef, deps, bucket)` ctor deep-links: a combined `"primary_secondary"` splits on the first `_`, a bare key is the secondary.
- **[`LeaderboardPageDeps`](LeaderboardPageDeps.java)** / **[`LeaderboardScreenMessages`](LeaderboardScreenMessages.java)** - the immutable policy bundle + the locale-free chrome the consumer implements (pre-built client-resolved `Message`s). Difficulty-tab + stat-column labels live on the deps, not the messages interface.
- **`.ui`** (shipped in this jar's pack): `Pages/ZigLeaderboardPage.ui` (860x700), `ZigLeaderboardRow.ui` (Rank/Player/Total/Score/Time/Plays), `ZigLeaderboardStatsRow.ui` (Rank/Player/#Col0..3/Plays), `ZigLeaderboardTab.ui`. Active tab/sort/view buttons are tinted from Java over `ZigButtons.ui @ZigTabBtnStyle`. Not gradle-compiled - smoke-test in-game.
- **[`LeaderboardLayoutAsset`](LeaderboardLayoutAsset.java)** + **[`LeaderboardLayout`](LeaderboardLayout.java)** + **[`LeaderboardLayoutConfig`](LeaderboardLayoutConfig.java)** - the pack-authorable layout (Pattern A, common-registered, `Server/ZiggfreedCommon/Leaderboard/`): `BoardId` + difficulty/party-size tab axes + stat columns, all label-KEYS resolved to client `Message`s via `Message.translation` in `toLayout(id)` (so the consumer builds `LeaderboardPageDeps` straight from the resolved record). A difficulty tab's `Id` IS the bucket prefix (must equal the preset id). Replaces a consumer's hardcoded `LeaderboardPageDeps` literals (Kweebec authored `Chase.json`). `MAX_STAT_COLUMNS`=4 still caps the rendered columns.
- **Tests**: [`LeaderboardTest`](../../../../../../test/java/com/ziggfreed/common/instance/leaderboard/LeaderboardTest.java) (engine-free) covers accrual, stat merge, `forBuckets`, and the sort comparators.
