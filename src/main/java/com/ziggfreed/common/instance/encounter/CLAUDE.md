# instance/encounter/ - the generic spawn-roster + wave director primitive

Router for `instance/encounter/`. PURE LOGIC, mod-agnostic, engine-free: no Store/Ref/World, no ids, no config, no logging - the consumer supplies the units (weight, tier, count) and the policy, calls these to decide WHAT spawns and WHETHER/HOW MANY, then does the engine spawn itself on the world thread. The generalized lift of Kweebec's `AiHunterController` roster + escalation bookkeeping (read that file for the source algorithm). Determinism is the contract: callers pass the seed and the clock; nothing here reads `Math.random()` or the wall clock.

- **[`SpawnUnit`](SpawnUnit.java)** - immutable `record(unit, weight, spawnTier, count)` over a generic `T` (an archetype asset, a role id, an enum). `eligibleAt(tier)`, `safeWeight()` (clamped `>= 0`), `safeCount()` (clamped `>= 1`). `of(unit, weight, spawnTier)` = count 1.
- **[`SpawnRoster`](SpawnRoster.java)** - immutable list of `SpawnUnit<T>` (copied on construction). `planRoster(tier, cap, seed)` = weighted-with-replacement fill to `cap` over units eligible at `spawnTier <= tier`, each pick adding `count` copies, deterministic via `new Random(seed)`; `weightedPick(tier, seed)` = one eligible pick (null if none). Generalizes `planRoster`/`weightedPick`/`addArchetype` exactly (zero-total weight falls back to uniform). The consumer prepends its own primary/forced seed unit if it wants one - that is policy, not in the roster.
- **[`EncounterDirector`](EncounterDirector.java)** - mutable per-round wave gate (`ConcurrentHashMap` trackers; world-thread caller, but safe). `canFire(ruleKey, nowMs, cooldownMs, maxFires)` = cooldown + max-per-round gate (`<= 0` disables either gate); `recordFire(ruleKey, nowMs)` stamps the clock + bumps the count; `allowedToSpawn(liveCount, hardCap, requested)` clamps a wave to `min(requested, hardCap - liveCount)` floored at 0 (the `maybeEscalate` cap guard); `fireCount(ruleKey)`; `reset()` at round start/end. The live count + clock are PARAMS so it stays pure - the engine read happens caller-side on the world thread.
- **Pack-authorable codec assets** (Pattern A, common-registered by `asset/FrameworkAssetRegistrar` - the asset FACE of the pure-logic primitives above): **[`MultiPhaseBossAsset`](MultiPhaseBossAsset.java)** + **[`MultiPhaseBossConfig`](MultiPhaseBossConfig.java)** (a multi-phase boss: phase roles + HP-fraction thresholds + per-phase adds + cues + optional per-phase HELPER-THROWABLE clusters `ThrowableClusterId`/`PhaseNThrowableCount`/`ThrowableRingRadius`/`ThrowableRespawnSeconds` the consumer's controller places + replenishes around the boss so survivors gather/throw them at a high-HP boss; `Server/ZiggfreedCommon/Bosses/`); **[`EncounterRuleAsset`](EncounterRuleAsset.java)** + **[`EncounterRuleConfig`](EncounterRuleConfig.java)** (a wave rule feeding `EncounterDirector`: free-string `Trigger`/`Placement` the consumer resolves + count/weight/cap/cooldown/maxPerRound; `Server/ZiggfreedCommon/EncounterRules/`). Both extend `AbstractKeyedAssetConfig` (defaults<pack<owner), ship ZERO common defaults (content is consumer pack JSON), and are consumed by Kweebec's `BossController` / `AiHunterController`. Lifted from kweebec's `BossAsset`/`SpawnRuleAsset`.
- **Tests**: engine-free; cover the weighted pick determinism (same seed -> same plan), tier eligibility, count-honoring fill, the cooldown/max-fire gate, and the `allowedToSpawn` clamp.
