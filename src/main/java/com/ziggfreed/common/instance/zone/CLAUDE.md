# instance/zone/ - co-op zone-hold objective primitive

Router for `instance/zone/`. Part of the reusable instance/encounter framework (see the parent `CLAUDE.md` blockquote). PURE logic, no engine deps - mirrors [`../encounter/EncounterDirector`](../encounter/EncounterDirector.java) (caller injects `nowMillis` + the live counts; this never reads a `Store`/`World` or the wall clock).

- **[`ZoneHoldTimer`](ZoneHoldTimer.java)** - a continuous "the required group must hold this zone TOGETHER for X seconds, RESET on break" timer. `update(occupants, required, nowMillis)` accrues while `occupants >= required` and resets the instant that breaks, latching `true` once the hold completes (resolve the objective on the first `true`). `isHolding()`/`remainingSeconds(now)`/`progress(now)` drive a HUD countdown; `reset()` is the explicit break. The CONSUMER owns the policy: WHO counts as an occupant (e.g. all-mobile vs whole-party, cocoon handling), WHERE the zone is (a center + radius occupancy test), and what completion does. Reusable for an extraction pad, a capture point, king-of-the-hill, a ward-defense hold.
- **Exemplar consumer**: Kweebec Nightmare's Chase extraction platform - `ChaseState` owns one `ZoneHoldTimer` (built from `RuleSet.extractionHoldSeconds()`), `ChaseMode.checkExtraction` feeds it the on-platform count vs the required survivor set (its `ExtractionMode` ALL_MOBILE / EVERYONE policy + cocoon rules) and marks the group escaped on completion; `NightmareHud` reads `remainingSeconds` for the "X/Y - Ns" line.
