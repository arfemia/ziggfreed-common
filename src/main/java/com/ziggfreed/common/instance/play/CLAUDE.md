# instance/play/ - the reusable Public/Party/Solo "Play" screen

Router for `com.ziggfreed.common.instance.play`. ONE generic, mod-agnostic page that turns "I picked a difficulty" into "how do I want to enter?" - the queue-mode chooser the consumer's NPC dialogue / command opens, superseding the old single-purpose `QueuePage`. A consumer holds ONE [`PlayModePageDeps`](PlayModePageDeps.java) and supplies only policy + pre-built `Message`s; nothing mod-specific leaks into common.

- **[`PlayModePage`](PlayModePage.java)** - the two-state screen (extends the lifted [`../../ui/toast/ToastablePage`](../../ui/toast/CLAUDE.md)). It MORPHS off the player's live-queue status (`LobbyService.currentQueueOf`):
  - **Chooser** (not queued): a chosen-difficulty header + the asset-driven mode cards (`#ModeCards`). Each card = an item-glyph icon (`ItemGrid #CardIcon.Slots`, the only Java-settable icon path) + a clickable name button + a one-line description. Picking a card routes through the consumer's [`PlayModeHandler`](PlayModeHandler.java): PUBLIC queues + the page reopens itself to the roster, PARTY opens the party manager (carrying the preset), SOLO launches immediately.
  - **Roster / ready** (queued): the live roster, an `X/Y` count, the queue state, and a Leave button. **The launch countdown ticks DOWN live** - a registered [`../../lobby/QueueListener`](../../lobby/CLAUDE.md) pushes the new `#Status`/`#PlayerCount` via `sendUpdate` on every `onChange` (fired each second by `MatchmakingQueue.onCountdownTick`), no reopen, no polling, no scheduler. The push updates ONLY the always-present template scalars (never the dynamically-appended roster rows -> no selector race) and is guarded by `ToastablePage.isDismissed()` (an unguarded stale push crashes the client - the same hazard the toast surface guards). Close keeps the queue reservation; Leave de-queues; `onDismiss` detaches the listener.
- **Asset-driven modes**: which modes are offered, their icon item-id, order, and label live in the consumer's `Server/<Mod>/Instances/*.json` ([`../preset/InstancePresetAsset`](../preset/InstancePresetAsset.java)'s `QueueModes` block) -> resolved [`../preset/QueueModeSet`](../preset/QueueModeSet.java) (per-preset gating, e.g. a co-op-only difficulty disables Solo). The mode CONTENT is NEVER baked in this library: `QueueModeSet.fallback()` is a neutral structural net only (all enabled, no glyph). [`../preset/QueueModeId`](../preset/QueueModeId.java) = the fixed identity (PUBLIC/PARTY/SOLO); its launch behaviour is the consumer's `PlayModeHandler`.
- **The DRY boundary**: the page imports no mod/lobby types. [`PlayModePageDeps`](PlayModePageDeps.java) carries the `LobbyService`, a `preset -> QueueModeSet` resolver, a `preset -> Message` name resolver, a lang-key resolver (for an authored `LabelKey` override), the `PlayModeHandler`, and the locale-free [`PlayScreenMessages`](PlayScreenMessages.java). [`PlayEventData`](PlayEventData.java) is the click codec (`pick`+mode / `leave` / `close`). Every `handleDataEvent` exit path sends a response.
- **Solo = launch now**: `PlayModeHandler.launchSolo` -> `LobbyService.launchSolo` (a private self-scoped queue + a 0/0 `LobbyConfig`, fires the launcher immediately off-lock; no fill wait, no strangers).
- **`.ui`**: `Pages/ZigPlayPage.ui` (frame + `#ModeCards`/`#RosterArea` toggled by `.Visible`) + `Pages/ZigPlayModeCard.ui` (icon + button + sublabel); reuses `Pages/ZigQueueRosterRow.ui`. Validate in-game (`.ui` is not gradle-compiled).
- **Consumer exemplar**: Kweebec's `experience/KweebecPlayMode` + `KweebecPlayScreenMessages`, `dialogue/OpenPlayAction` (the `{ "Play": "<preset>" }` dialogue sugar), `Server/KweebecNightmare/Instances/*.json`. Plan: [[kweebec-playmode-public-party-solo]].
