# instance/arena/ - pack-authorable arena layouts

Router for `com.ziggfreed.common.instance.arena`. A mod-agnostic, pack-authored ARENA definition (the SPATIAL layout of a minigame battlefield) so any minigame authors team spawns / objective points / pickup spots as DATA instead of Java literals. The mod-root `CLAUDE.md` instance-experience section is the wider context.

- **[`ArenaDefinitionAsset`](ArenaDefinitionAsset.java)** - the pack-authorable arena (Pattern A, common-registered by `asset/FrameworkAssetRegistrar`, `Server/ZiggfreedCommon/Arenas/*.json`, one arena per file, PascalCase filename). Fields: `Id`/`Name` (echo the filename, no-op setters), `InstanceName` (the worldgen `instance.bson` to spawn), `Tags` (`String[]`), `TeamSpawns` (a list of teams, each a `{ "Anchors": [ {X,Y,Z,Yaw}, ... ] }` group), `ObjectiveAnchors` (`[ {Id,X,Z,Radius} ]`, radius defaults 4.0), `PickupAnchors` (`[ {X,Z} ]`). The codec IS the schema authority (same CODEC on pack + owner layers); list-of-objects use `ArrayCodec` over nested `BuilderCodec`s, the same idiom as `LeaderboardLayoutAsset`'s tab axes. Every `KeyedCodec` key is PascalCase (the ctor throws at static init otherwise). Accessors return small immutable records (`Anchor {x,y,z,yaw}`, `ObjectiveAnchor {id,x,z,radius}`, `PickupAnchor {x,z}`); an absent list yields an empty list. `StructureRoll` is intentionally OMITTED (see followups) - roll-in structures already have the generic `world/WeightedPrefabPlacementAsset` table.
- **[`ArenaDefinitionConfig`](ArenaDefinitionConfig.java)** - the `defaults < pack < owner` fold authority (extends `asset/AbstractKeyedAssetConfig`, holds the asset itself as the resolved type since it has clean accessors). `getInstance()` + `arenas()` (stable id order) / `byId(id)` (owner > pack > default, else null) / `byTag(tag)` (case-insensitive). Ships ZERO jar defaults (a consumer with a baseline floor calls `loadDefaults` once); read AFTER `LoadedAssetsEvent` has fired. The fold maps are concurrent (lock-free reads); no engine call here, so any thread is fine to read.
- **Registration**: `asset/FrameworkAssetRegistrar.registerAll` registers the `Arenas` store + the `LoadedAssetsEvent` merge into `ArenaDefinitionConfig` (no-mapper `AssetMergeAdapter.layer`, folding the asset directly). A consumer authors arenas there and reads them back through the config; it must NOT re-register the class.
