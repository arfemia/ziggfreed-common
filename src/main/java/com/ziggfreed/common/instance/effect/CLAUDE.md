# instance/effect/ - generic EntityEffect applier primitives

Router for `com.ziggfreed.common.instance.effect`. Mod-agnostic, config-free helpers for applying Hytale `EntityEffect`s to an entity: a timed apply and an idempotent per-entity band swap. Lifted from Kweebec's `AiHunterController.applySpeed` (pace-band swap) and `KweebecDamageSystem.applyStunVisual` (timed apply); the consumer supplies all ids/policy (no hardcoded ids, no i18n). **All methods are world-thread only** (they read/mutate an `EffectControllerComponent`); the CALLER guarantees the thread, the service never hops. Every engine touch is try-guarded to a no-op (`false` / `NONE`), so a not-yet-registered effect, a bad ref, or a missing controller never throws into the caller.

- **[`EffectBand`](EffectBand.java)** - immutable `record (int bandIndex, @Nullable String effectId, boolean oneShot)`. A null/blank `effectId` is the baseline / "clear" rung. `hasEffect()`, plus `persistent(idx, id)` / `baseline(idx)` factories. `oneShot` is advisory metadata only (the ladder does not act on it).
- **[`EffectBandLadder`](EffectBandLadder.java)** - an ordered `List<EffectBand>` parallel to a numeric scale, owning both the band cutoffs and the effect ids. Build via `ofParallel(double[] bands, String[] effectIds[, oneShot])` (the historical parallel-arrays shape) or `of(double[] bands, List<EffectBand>)`. `nearestBandIndex(double value)` snaps a live value to the nearest rung (ties pick the lower index, generalizing Kweebec's `nearestBandIndex`); `effectIdFor(int bandIndex)` / `bandAt(int)` read the rung (null for baseline / out-of-range). Static `nearestBandIndex(double[] scale, double value)` for a consumer keeping its own scale. Pure logic, no engine calls, no world-thread requirement; immutable, safe as a static.
- **[`EntityEffectService`](EntityEffectService.java)** - static utility, the core lift.
  - `applyTimed(ref, effectId, durationSeconds, overlap, CommandBuffer)` / `applyTimed(ref, ..., Store)` - resolve the effect id, fetch the `EffectControllerComponent` off the accessor, `addEffect(ref, fx, duration, overlap, accessor)`; returns `false` on any miss. Both overloads delegate to one private form over the shared `ComponentAccessor` supertype (Store + CommandBuffer both implement it, and `addEffect` itself takes an accessor).
  - `applyBand(ref, ladder, targetBandIndex, currentlyAppliedEffectIndex, Store)` - idempotent swap: `removeEffect` the old index (skipped when `NONE`), add the target band's effect, RETURN the new applied effect index for the CALLER to track per-entity (the service holds NO state, mirroring `HunterUnit.appliedEffectIndex`). Returns `EntityEffectService.NONE` (`Integer.MIN_VALUE`) for a baseline/clear rung or any degrade. Caller keeps its own "band == applied -> skip" guard to avoid per-tick churn.
  - `NONE` = `Integer.MIN_VALUE`, the engine's "none applied" sentinel. No caching (resolves per call); wrap an `AssetIndexCache` if a hot path wants it. No logging (hot per-tick + unit-JVM-safe).
