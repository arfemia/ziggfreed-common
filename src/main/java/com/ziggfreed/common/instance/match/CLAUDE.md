# instance/match/ - the match setup + outcome primitives

Router for `com.ziggfreed.common.instance.match`. The mod-root `CLAUDE.md` instance-experience section is the wider context. Pure, mod-agnostic match-shaping helpers a consumer wires from its `experience/` package; nothing mod-specific leaks into common.

- **[`TeamSplit`](TeamSplit.java)** - the team balancer. PURE, engine-free (`java.util` + `UUID` only, no Hytale imports, no RNG, deterministic), so a caller may run it anywhere - no Store/Ref, no world-thread requirement, nothing to try-guard. `split(roster, partyBlockSizes, teamCount, teamSize)` carves the online-first roster (party blocks at the head, sized by `partyBlockSizes` in roster order; the rest are solo fillers) into `teamCount` teams of up to `teamSize`, keeping each party block WHOLLY on one team (largest block first, ties by block order, onto the emptiest team) then filling with solos to balance (least-filled team first, roster order). A block bigger than `teamSize` is CLAMPED, never thrown: the prefix that fits is seated, the overflow demotes to fillers. Defensive on empty roster / `teamCount <= 0` / `teamSize <= 0` / mismatched totals (returns the empty-team shape, no exception). `assign(...)` is the same split flattened to a `UUID -> teamIndex` (0-based) map (unseated surplus players are absent).
- **Win-condition resolver** (sibling-built) - the outcome side of a match lives here too: given the per-team / per-player scores it decides who won (last-team-standing / highest-score / objective / time). See its own class doc when added; it is the natural peer to `TeamSplit` (setup -> play -> resolve).
