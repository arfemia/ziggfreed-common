# dialogue/ - the generic branching NPC dialogue engine

Router for `com.ziggfreed.common.dialogue`. A self-contained, mod-agnostic dialogue runtime lifted from hyMMO (which is being re-integrated as its first EXTENDER). A consumer builds ONE [`DialogueEngine`](DialogueEngine.java) via the builder, supplies a few seams, and opens [`page/DialoguePage`](page/DialoguePage.java); the `.ui` ships once here and resolves client-side.

- **PARADIGM: per-consumer engine, NOT a shared mutable registry.** `DialogueEngine.builder()` pre-seeds the generic action/condition/sugar types, the consumer `.action(...)`/`.condition(...)`/`.router(...)`s its own, and `.build()` assembles the codec graph ONCE before any decode - so there is no registration race and one mod's domain types never leak into another's. The MMO registers quest/reward/gate types; a minigame builds with generics only.
- **[`DialogueAction`](DialogueAction.java) + [`DialogueCondition`](DialogueCondition.java) are BOTH `Type`-discriminated polymorphic lists** (`Conditions` is an array of `{Type:...}` objects, symmetric with `Actions`). Generic actions: `Goto`/`Close`/`SetFlag` + the carriers `Talk`/`OpenPage`. Generic conditions: `Flag`/`NotFlag`. The dispatch `CodecMapCodec`s are engine-instance-owned (assembled in [`DialogueEngine.Builder.build`](DialogueEngine.java)); each subtype carries its own field `BuilderCodec` (PascalCase keys, guarded by [`DialogueEngineTest`](../../../../../../test/java/com/ziggfreed/common/dialogue/DialogueEngineTest.java)).
- **One registration binds schema + behavior + presentation + sugar so they can't drift**: a [`DialogueActionType`](DialogueActionType.java) carries the codec + a [`DialogueActionHandler`](DialogueActionHandler.java) + an optional [`DialogueOptionStyle`](DialogueOptionStyle.java) kind + an optional [`DialogueSugarExpander`](DialogueSugarExpander.java); a [`DialogueConditionType`](DialogueConditionType.java) carries the codec + a [`DialogueConditionEvaluator`](DialogueConditionEvaluator.java). The executor dispatches by a handler MAP (not `instanceof`), which is what lets a consumer add an action without editing the engine.
- **The opaque per-player handle is [`DialogueContext`](DialogueContext.java)** (`DialogueExecContext` adds dialogue/node/option). The engine only reads the generic flag store + forwards the context to registered evaluators/handlers; a consumer's evaluator reaches its domain state via `payload(Class)`. Build it with [`page/SimpleDialogueExecContext`](page/SimpleDialogueExecContext.java) inside a [`page/DialogueContextFactory`](page/DialogueContextFactory.java). Flag storage is the consumer's [`DialogueFlagStore`](DialogueFlagStore.java) (MMO: a player component; minigame: in-memory per-round).
- **Seams**: [`DialoguePageRouter`](DialoguePageRouter.java) (the `OpenPage` target -> open a page, default NONE), [`NpcNameProvider`](NpcNameProvider.java) (header name, default NONE), [`DialogueHeaderAnnotation`](DialogueHeaderAnnotation.java) (the one-line hint row, default NONE), [`i18n/DialogueI18n`](i18n/DialogueI18n.java) (namespace prefix + en-US `hasKey` probe; the ready-made [`I18nModuleDialogueI18n`](i18n/I18nModuleDialogueI18n.java) is zero-config). All bundled into a [`page/DialoguePageDeps`](page/DialoguePageDeps.java) built once at startup.
- **[`template/DialogueTemplateResolver`](template/DialogueTemplateResolver.java)** ports the `extends`/`params`/`nodeOverrides`/`extraNodes` DSL + the engine's sugar (run last). **The pruner is DOMAIN-AGNOSTIC**: a Start candidate / option carrying a `"PruneIfEmpty": "<paramName>"` marker is dropped when the param is empty/absent (the quest-specific predicate is gone), then unreachable nodes are dropped. Uses common's own dialogue-scoped [`template/JsonTemplateUtil`](template/JsonTemplateUtil.java).
- **[`validate/DialogueStructureValidator`](validate/DialogueStructureValidator.java)** is the STRUCTURAL audit only (missing/dangling start, dangling Goto, unreachable, empty-after-resolve) returning neutral `Issue`s; a consumer that registers domain actions runs its OWN pass for their refs and merges.
- **[`page/DialoguePage`](page/DialoguePage.java)** is the generic page (extends the engine `InteractiveCustomUIPage`, no toast base). Every exit path sends a response (the client spins otherwise). The `.ui` lives in `src/main/resources/Common/UI/Custom/` as `Pages/ZigDialoguePage.ui` + `Pages/ZigDialogueOptionRow.ui` + `Common/ZigButtons.ui` + `Common/ZigFrames.ui` (neutral `Zig*` names so they never collide with a consumer's own `.ui`); all textures are base-game. The `.ui` is NOT gradle-validated - smoke-test in-game.
