# entity/ - player/entity model, puppet, and item-prop primitives

Router for `com.ziggfreed.common.entity`. Mod-agnostic helpers for mutating a rendered entity or
spawning a transient one.

- **[`PlayerModelService`](PlayerModelService.java)** - swap a PLAYER's rendered model and restore it, mirroring the engine `builtin.model.pages.ChangeModelPage` (`ChangeModel` / `ResetModel`). `apply(ref, store, modelId, scale)` resolves the `ModelAsset` (missing/invalid = no-op, warned once per id), builds a scaled `Model` via `Model.createScaledModel` (scale floored to a positive value so the engine's `scale > 0` guard never throws), and replaces the immutable `ModelComponent`; returns `true` only when applied. `restore(ref, store)` rebuilds the player's CORRECT cosmetic model from `PlayerSkinComponent` via `CosmeticsModule.get().createModel(skin.getPlayerSkin())` then `setNetworkOutdated()` - NOT a cached `Model` (a cached one holds a stale reference); idempotent, safe on any player (no-op without a skin component). `modelExists(modelId)` for early preset validation. **WORLD-THREAD ONLY** (the caller guarantees the thread for every `Store`/`Ref` read + `putComponent`; this primitive does not hop itself); every engine call is try-guarded so a bad ref / missing asset degrades to a no-op and never throws into the caller. `restore` is also the revert path for a `ModelSwap` self-hide route (see `PlayerPuppetService` below - it does not duplicate this).
- **[`PlayerPuppetService`](PlayerPuppetService.java)** - the "mount the player, hide their body, show a puppet performing the work instead" presentation mechanism, lifted config-free out of a consumer's in-game P0 spike harness (RPG Stations' `puppetspike.PuppetSpikeService`; the design doc is `.claude/research/raw/rpg-stations-puppet-presentation-design-2026-07-22.md`). `spawn(accessor, PuppetSpawnRequest)` clones a source player's live `PlayerSkin` (`PlayerSkin` copy ctor -> `PlayerSkinComponent` -> `CosmeticsModule.createModel`) onto a new networked, `NonSerialized`, `Visible` entity at a caller-resolved position/yaw, with optional held-item mirroring (a bare `InventoryComponent.Hotbar` slot 0 - `InventorySystems.SyncEquipmentSystem` has no player-type gate) and an optional `ActiveAnimationComponent` pre-seed on the spawn holder (the render-guaranteed route: any viewer whose tracker marks the puppet newlyVisible on a LATER tick auto-catches-up via the engine's own `ModelSystems.AnimationEntityTrackerUpdate`, independent of when a direct animation packet lands). `PuppetSpawnRequest` is an immutable builder (`sourceRef`/`position`/`yawRadians`/`mirrorHeldItem`|`heldItemIdOverride` mutually-exclusive last-set-wins/`initialAnimation(slot, clipId)`), mirroring `cast.HitResolver.HitRequest`'s shape. `despawn(puppetRef, Store|CommandBuffer)` (two overloads - the interface's own `removeEntity` needs a `Holder` neither caller has at despawn time) and `playAnimation(accessor, puppetRef, slot, itemAnimationsId, clipId, sendToSelf)` (the single-shot primitive; the caller owns its own repeat-beat cadence, mirroring a station engine's per-swing timer convention) round out the surface. Plus the paired SCALE self-hide primitive - `hideByScale(accessor, ref[, nearZeroScale])` returns the PRIOR `EntityScaleComponent` scale (`null` = none existed, so revert REMOVES the component rather than resetting to `1.0`) as the revert payload for `revealByScale(accessor, ref, priorScale)`; this is the in-game-CROWNED `Hide.Route: "Scale"` mechanism (confirmed to fully hide a player's own rendered body, including the held item, in both first- and third-person - unlike a `ModelComponent` swap). A `ModelSwap`-route hide reuses `PlayerModelService.restore` for its revert; this class does not duplicate it. Every spawn-side method takes a `ComponentAccessor<EntityStore>` (the interface both `Store` and `CommandBuffer` implement, the same seam `cast.HitContext` uses) so an interaction-handler-locked caller (needs a `CommandBuffer`) and a live-`Store` caller both work through ONE method. Plus two pure spatial helpers (`offsetPosition`, `yawRadiansFromDegrees`) kept primitive-typed for unit-testability. **Deliberately narrow scope**: this class owns the spawn MECHANISM + the scale hide/reveal PAIR only - offset/yaw resolution against a station's own anchor concept, which `Look`/`Hide` route an author picked, and the swing-beat cadence all stay caller (consumer mod) policy. WORLD-THREAD ONLY; every engine call is try-guarded to a no-op/null, never a throw.
- **[`ItemPropEntityService`](ItemPropEntityService.java)** - spawns/despawns a static, network-replicated, pickup-immune, physics-free PROP entity representing an item at a fixed position (a placed-input display, a shop pedestal, a trophy case, ...). Lifted config-free out of a consumer's placed-input display (RPG Stations' `station.StationCustodyDisplay`), itself copied verbatim from the engine's own sanctioned admin "Entity Spawn Page" Items-tab exemplar. Two routes picked by `Item#hasBlockType()`: a block-shaped item spawns a real `BlockEntity` (the actual block model, with the exemplar's own `BLOCK_ENTITY_BASE_SCALE` doubling); everything else spawns a bare `ItemComponent` prop with `setOverrideDroppedItemAnimation(true)` (the generic "dropped item minus physics" shape). Both routes carry `PreventPickup`/`PreventItemMerging`/`PropComponent` and are `NonSerialized` (never survives a restart, no orphan-reconcile needed). Two-phase API: `buildHolder(accessor, itemId, position, rotation, scale)` returns an un-added `Holder` so a caller can add its OWN components (e.g. a press-F retrieve interaction) before committing via `spawn(accessor, holder)`; a one-call `spawn(accessor, itemId, position, rotation, scale)` convenience composes both for a caller with no such customization. `despawn(propRef, Store|CommandBuffer)` mirrors `PlayerPuppetService`'s own two-overload shape. **NOT YET CONSUMED**: `StationCustodyDisplay` still carries its own copy of this mechanism as of this lift (rewiring it onto this primitive is a follow-up leg, not done here, to avoid touching rpg-stations' actively-smoke-tested custody/display code in the same change) - the two implementations are mechanism-identical by construction (this class IS the verbatim lift), so consuming it later is a pure delegation, not a behavior change.

**Lift-review note (station entity-anchor spawn, SKIPPED this leg)**: RPG Stations' `station.StationEntityMountController#spawnAnchor` (the `Hold.Mount.Surface: "Entity"` standing-work-mount anchor) was evaluated and NOT lifted - its own class javadoc self-describes as "a phase-2 SPIKE component set, iterate in-game", the feature is flagged in-game-unverified, and this same session's R6 fix wave just moved the shipped Anvil OFF this route back onto the proven effect-mode hold pending further verification. Lifting an actively-unstable, still-changing component shape into a "generic" common primitive would freeze a spike shape prematurely; revisit once the standing-mount route is proven and its component list stabilizes.
