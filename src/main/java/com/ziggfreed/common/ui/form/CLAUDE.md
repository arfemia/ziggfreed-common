# ui/form/ - the generic settings-form engine

Router for `com.ziggfreed.common.ui.form`. A mod-agnostic engine that turns an ordered [`FieldSpec`](FieldSpec.java) list into appendable, bindable, collectible settings rows for any consumer's admin/config page (the mmo-mob-scaling admin page is the first consumer). No i18n keys, no consumer content types, no persistence policy live here - specs, label resolution, and writing the result somewhere (typically `util/JsonOverrideWriter`) all stay in the consumer.

- **[`FieldKind`](FieldKind.java)** - the ten field kinds: `TEXT`/`NUMBER`/`CHANCE`/`INT`/`CSV` (leaf-bearing controls), `TOGGLE` (instant on/off, never collected), `TRISTATE`/`DROPDOWN` (dropdown-backed leaves), `HEADER`/`NOTE` (label-only, never collected).
- **[`FieldSpec`](FieldSpec.java)** - one immutable field (id, kind, optional dotted-PascalCase leaf path, an opaque label key, and DROPDOWN's literal entries), built only via its static factories (`FieldSpec.text(id, leafPath, labelKey)`, etc).
- **[`SettingsForm`](SettingsForm.java)** - one instance per form; owns the spec list plus a raw `Map<String, String>` value cache (default `""`).
  - `buildRows(cmd, events, containerSel, labels)` appends one row per spec (`containerSel + "[" + i + "]"`), painting the label + current value and wiring the live binding.
  - `applyValues`/`applyValue(cmd, containerSel[, fieldId])` push cached values into ALREADY-BUILT rows (no append, no rebinding) for a partial `sendUpdate` refresh.
  - `cache(fieldId, value)` is the `ValueChanged` sink (false + no-op for an unknown id or a null value); `value(fieldId)` reads it back (`""` default); `seed(Map)`/`seedValue` pre-load the cache (e.g. from a loaded config).
  - `collectLeaves(blankIsInherit)` parses the cache into a [`FormResult`](FormResult.java): a spec-order `Map<String, Object>` leaf map (a null value means "remove this leaf", ready for `JsonOverrideWriter.setLeaves`) or the first validation failure.
- **Event shapes** (both wired by `buildRows`, both routed by the CONSUMER's `handleDataEvent`): a TEXT-ish/TRISTATE/DROPDOWN control binds `ValueChanged` as `EventData.of("Action", "field").append("Field", id).append("@Value", <selector>.Value)`; a TOGGLE binds `Activating` as `EventData.of("Action", "press").append("Field", id)` (via `SettingsUiUtil.bindButton`). The engine never flips a toggle's own cached value on `"press"` - the consumer decides what a press means (e.g. read the current value, flip it, call `cache`) and re-renders.
- **`blankIsInherit`** governs TEXT/NUMBER/INT uniformly: `true` = a blank field means "inherit the layer below" (null leaf); `false` = a blank NUMBER/INT is a validation error, a blank TEXT becomes an explicit `""` leaf. CSV/TRISTATE/DROPDOWN ignore the flag (see `collectLeaves`'s javadoc for their own blank rule).
- **Row template child-id contract** (every `Pages/ZigForm*Row.ui` must carry): `#Title` is the row label for every kind except `NOTE` (which labels `#Note` instead); `#Field` is the TEXT-ish control; `#Dropdown` is the TRISTATE/DROPDOWN control; `#Toggle` is the toggle button (its OWN inner `#Label`, driven by `SettingsUiUtil.setToggle`, never collides with the row's `#Title`). Five templates ship in `Common/UI/Custom/Pages/`: `ZigFormFieldRow.ui`, `ZigFormDropdownRow.ui`, `ZigFormToggleRow.ui`, `ZigFormHeaderRow.ui`, `ZigFormNoteRow.ui` (not gradle-compiled; smoke-test in-game).
