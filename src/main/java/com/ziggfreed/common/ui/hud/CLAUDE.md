# ui/hud/ - keyed in-world HUD overlay base + layout value

Router for `com.ziggfreed.common.ui.hud`. Mod-agnostic in-world HUD-overlay primitives lifted from the MMO Skill Tree's `ui/hud/` (the MMO is the shared-file owner; it re-parents its own `MmoHud` onto the base here and keeps only its own conveniences).

- **[`KeyedCustomHud`](KeyedCustomHud.java)** (RPG Stations extraction, phase 1 leg 1): the shared base every custom in-world HUD extends, `extends` engine `CustomUIHud`. Owns a `HudPosition`-driven anchor on a single root element (fixed-height via `toAnchor` or content-height via `toAnchorContentHeight`, see `usesContentHeight()`), two push-throttle strategies (`dueForPush`/`markPushed` for a tick loop that gates BEFORE reading state, `acquirePush` for a CAS atomic acquire), and the native keyed register/lookup/broadcast over Update 5's `HudManager` (`get`, `forEachOnlineHud`, `refreshPositionForAllOnline`). A concrete HUD supplies only its identity + layout (`rootSelector()`, `panelWidth()`, `panelHeight()`, `configuredPosition()`, `updateIntervalMs()`), calls `applyConfiguredPosition` in `build()`, and pushes via one of the throttle helpers. **Mod-agnostic on purpose**: player lookup (`resolvePlayer`) and world resolution (the "any alive world" broadcast fallback) are self-contained here (no consumer world-lookup utility required), and logging is a locally-guarded `ZiggfreedCommonPlugin.LOGGER` call (unit-JVM safe, mirrors the pattern every other primitive in this jar uses - no consumer logging facade required either). A consumer with its OWN HUD base (the MMO's `MmoHud`) re-parents onto this and keeps adding only its own conveniences; a consumer with no base of its own (RpgStations' `StationSummaryHud`) extends this directly.
- **[`HudPosition`](HudPosition.java)**: the mod-agnostic layout value - a vertical `AnchorEdge` (TOP/BOTTOM/CENTER) + horizontal `HorizontalEdge` (LEFT/CENTER/RIGHT) + pixel offsets. `toAnchor(w, h)` / `toAnchorContentHeight(w)` always set `Width` (and `Height` for the fixed form) and pin EXACTLY ONE edge per axis so an offset moves the panel instead of being neutralized by an opposing stretch - including the CENTER-horizontal case (`Left` only when `offsetX != 0`, never `Horizontal`, which is a LEFT+RIGHT inset that would stretch the panel to fill the screen minus the offset). Carries NO consumer-specific defaults (`parse`/`isValidPreset` cover the config-facing named-corner-preset form); a consumer HUD supplies its own default position (e.g. `MyHud.defaultPosition()`).
- **Consumers**: the MMO's `AbilityCooldownHud`/`QuestTrackerHud`/`SessionSummaryHud` (via the thin `MmoHud` shim) and, going forward, RpgStations' `StationSummaryHud` (extends `KeyedCustomHud` directly, no MMO dependency).
- **Conventions**: `@Nonnull`/`@Nullable`; the guarded `warn(...)` here is `KeyedCustomHud`-local (not a shared facade - this jar's other primitives each guard their own `ZiggfreedCommonPlugin.LOGGER` call inline); no em-dashes.
