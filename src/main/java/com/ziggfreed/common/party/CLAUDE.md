# party/ - the reusable party + searchable-invite primitive

Router for `com.ziggfreed.common.party`. A net-new, mod-agnostic party system (the lobby's social front-end; no native Hytale party primitive exists), mirroring `lobby/`'s shape 1:1. A consumer holds ONE [`PartyService`](PartyService.java) and supplies only policy + pre-built `Message`s; nothing mod-specific leaks into common.

- **[`PartyService`](PartyService.java)** owns the live [`Party`](Party.java)s + the global single-party-per-player reservation (`partyOf`, the lobby `queuedTo` twin) + a daemon scheduler for invite expiry. `getOrCreate`/`invite`/`accept`/`decline`/`kick`/`leave`/`disband`/`transferOwner`/`setPrivate` (results [`InviteResult`](InviteResult.java)/[`AcceptResult`](AcceptResult.java)), `partyOf`/`isInParty`/`pendingInvitesFor`. All mutations `synchronized`; ALL delivery is **packet-only** (`Universe.getPlayer` + `Notify`/`EventTitles`, NEVER a Store/Ref read), every `Message` from the consumer's [`PartyMessages`](PartyMessages.java) (locale-free). Invite expiry uses an injected `LongSupplier clock` (test seam). One [`PartyListener`](PartyListener.java) hook for UI refresh.
- **[`Party`](Party.java)** - the ordered member set (owner first) + pending invites + the owner-settable `privateLobby` flag (queue privacy; default public); reads + package-private mutators `synchronized`. [`PartySnapshot`](PartySnapshot.java) (carries `privateLobby`) is the only UI render source (usernames resolved live at render time, never stored). [`PartyConfig`](PartyConfig.java) = maxSize / inviteTimeoutSeconds / ownerOnlyInvite.
- **[`page/PartyInvitePage`](page/PartyInvitePage.java)** - the two-tab screen (Coven roster + incoming invites with Accept/Decline / footer; Invite = a searchable live online-player list via `Universe.getPlayers()` + `ValueChanged` on `#SearchField`). Built from an immutable [`PartyPageDeps`](page/PartyPageDeps.java) (service + locale-free [`PartyScreenMessages`](page/PartyScreenMessages.java) chrome + an optional [`PartyQueueHandler`](page/PartyQueueHandler.java) for the Queue button). The owner footer carries a **public/private pill** (`#PrivacyBtn`, its label = the current state, click flips it via `setPrivate`). Extends the lifted [`../../ui/toast/ToastablePage`](../../ui/toast/CLAUDE.md): an invite outcome + a privacy change show an in-page toast (the off-page `Notify` feed is hidden behind the open menu). Reuses `$F.@ZigDecoratedFrame` + `$Z.@ZigOptionBtnStyle`; `.ui` `Pages/ZigPartyPage.ui` + `Pages/ZigPartyRow.ui` ship here. Every `handleDataEvent` exit path sends a response.
- **Queues as a UNIT**: the Queue button routes through the consumer's `PartyQueueHandler` -> `LobbyService.queueParty` (atomic group-join; see [`../lobby/CLAUDE.md`](../lobby/CLAUDE.md)). Public (default) = shared preset queue (backfill); private = `QueueKey.privateQueue(gameId, presetId, partyId)` when the owner sealed the party (the consumer reads `Party.isPrivate()` and passes the party id as the scope). **`PartyQueueHandler.queue` + the `PartyInvitePage` 5-arg ctor carry a `@Nullable String presetId`** (the difficulty chosen upstream on the [`../../instance/play/`](../../instance/play/CLAUDE.md) Play screen's Party card) so the group queues at THAT preset, not a hardcoded default; a null preset (standalone `/party`) falls back to the consumer's default.
- **Consumer exemplar**: Kweebec's `experience/KweebecParty` + `KweebecPartyMessages` + `KweebecPartyScreenMessages`. Plan: [[proud-wondering-tiger]].
