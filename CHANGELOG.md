# Changelog - Ziggfreed Common

The dev changelog for the shared, mod-agnostic Hytale primitive library. Newest first. No em-dashes.

## 1.4.0
A reusable cast / ability RUNTIME layer (`cast/` + `cast.step/`), lifted config-free from a consumer, so a mod that fires abilities reuses ONE battle-tested set of runtime seams instead of re-deriving the generic step-dispatch kernel, the on-hit dispatch, the per-hit resolver, the armed-state store, a guarded observer registry, ray/block targeting, and the per-world tick partition. All additive over 1.3.0; a consumer picks up only what it calls, and the layer ships EMPTY of content and of any step vocabulary (every built-in type, step handler, and drain system is consumer policy; the kernel is parameterized over the consumer's own context / step / result types). The `cast/` surface is ADDITIVE-ONLY once frozen.

- New: `cast.step` - the generic ordered step-dispatch KERNEL, parameterized over a consumer's OWN context `C`, step `S`, key `K`, and result `R` (common defines no step type of its own). `StepHandler<C,S,R>` is the handler SAM (`R execute(C, S)`); `StepRegistry<K,C,S,R>` is a per-consumer `register` / `@Nullable get` map that ships EMPTY, is last-write-wins + null-key-safe, and offers a protected backing-map constructor so an enum-keyed subclass can use an `EnumMap`; `StepSemantics<C,S,K,R>` is the tiny consumer-supplied contract (`stepsOf` / `keyOf` / `isSuccess` / `successResult` / `onMissingHandler`) that adapts a consumer's existing spec / step / result methods to the walk with NO `implements` on those types; `CastKernel<C,S,K,R>.run(ctx)` walks the steps in order, dispatches each by key, short-circuits on the first non-success result, succeeds on an empty chain, and defers the missing-handler behavior (logging + error result) to the consumer's `onMissingHandler`. Byte-parity with a hand-written dispatch loop. Unit-tested (`CastKernelTest`: order, short-circuit, missing-handler hook, null key, empty chain; `StepRegistryTest`: register/get/empty, last-write-wins, null-key, enum-backed subclass).
- New: `cast/OnHitRegistry` - a PER-CONSUMER on-hit builder registry (news up your own; two mods sharing this jar never see each other's type registrations). `register(type, OnHitBuilder)` keys case-insensitively, last write wins; `fromParams(Object|Map, sourceRef, sourcePlayerId)` converts an `onHit` JSON shape - a single object OR a list chained in declaration order via `andThen` (NO_OP entries skipped) - into a `BiConsumer<Store, Ref>`. A PARALLEL `registerAction(type, HitActionBuilder)` + `actionFromParams(...)` path (separate table) produces a `HitAction` instead, for a callback that needs the richer `HitContext` (a `CommandBuffer` accessor, damage amount, hit position); a type registered on one table is invisible to the other. An unknown type or a builder that throws degrades to the shared `NO_OP` / `NO_OP_ACTION` sentinel with a guarded FINE log. Ships with ZERO registered types. Unit-tested (`OnHitRegistryTest`: single/list dispatch, declaration-order chaining, NO_OP identity for null/malformed/unknown/empty, builder-throws degrade, case-insensitive keying, two-instance isolation, and the parallel HitAction table not colliding with the BiConsumer table).
- New: `cast/HitContext` + `cast/HitAction` - the evolvable successor to a raw `BiConsumer<Store, Ref>` on-hit hook. `HitContext` is an immutable, FIELD-ADDITIVE carrier (accessor typed `ComponentAccessor<EntityStore>`, so a live `Store` AND a `CommandBuffer` both fit - the deferred-mutation seam - plus target / source / sourcePlayerId / damageAmount / position / cause), built via a builder so a new field never breaks an implementer. `HitAction` is `void onHit(HitContext)` with a `NO_OP` and a null-dropping `andThen` chain.
- New: `cast/HitResolver` - a stateless per-already-resolved-target hit routine. `applyHit(store, HitRequest, target, hitPos)` runs the damage dispatch under an opaque `UnaryOperator<Runnable>` origin decorator (wrapping EXACTLY the dispatch), then the on-hit hook (a `HitAction` OR a bridged `BiConsumer<Store, Ref>` fired with no context allocation), then an opaque per-hit sound sink (`BiConsumer<Store, Vector3d>`) and an optional per-hit particle via `ModelParticleService`; it returns whether the dispatch completed without throwing (logged WARN or FINE per the request). The `DamageSink` builds the engine `Damage` itself from the request's source / cause / amount, so common constructs no `Damage` and every knob (source / cause / amount / fail labels / warn-vs-fine / per-hit sound + particle) rides an immutable `HitRequest` builder. All logging routes through the guarded common LOGGER. Unit-tested (`HitResolverTest`: the origin-decorator wrap point, the HitAction + BiConsumer-bridge dispatch, last-setter-wins, throwing-dispatch returns false, throwing-onHit is isolated, sound gated on a position, and both `Store` / `CommandBuffer` accessor forms round-tripping through `HitContext`).
- New: `cast/ObserverRegistry<E>` - a PER-CONSUMER guarded observer registry: `register(Consumer<E>)` + `fire(E)` over a `CopyOnWriteArrayList` with a PER-LISTENER try/catch, so one misbehaving listener neither throws out of `fire` nor blocks the listeners after it. Optional label constructor tags the guarded warn. Unit-tested (`ObserverRegistryTest`: fire-in-order, throwing-listener isolation, empty-fire no-op).
- New: `cast/CastParams` - static param-coercion helpers (`numberOr` -> `Number`, `stringOr` -> nullable `String`, `boolOr`) for a `Map<String,Object>` params bag; a mistyped value falls back, never throws. Pure, no engine. Unit-tested (`CastParamsTest`).
- New: `cast/ArmedStateStore` - a PER-CONSUMER store of per-caster transient combat state: next-hit damage multipliers / flat damage / on-hit chains keyed `(casterId, slot)`, plus a parallel invulnerability window. `armNextHit`/`armMultiplier`/`peekArmedMultiplier`/`consumeArmed` (a composite across all live slots: multipliers multiply, flats sum, on-hit consumers chain, and each live slot's OPAQUE `attribution` payload is exposed in encounter order via `ArmedState.attributions` for the consumer to fold) + `armInvulnerability`/`isInvulnerable`/`peekInvulnerability`/`clearInvulnerability` + `clearPlayer`, with expiry-on-read pruning. Each arm / invulnerability call carries a single `@Nullable Object attribution` the consumer attaches (its own attribution record, a source tag, ...); common never inspects it, so no consumer-domain vocabulary lives in this class. The payload-free `armMultiplier(UUID, double, long)` convenience overload is retained, and `ArmedState` / `InvulnerabilityState` are immutable (final fields + constructor). The CALLER supplies the UUID key and the store NEVER derives it - the consumer's caster-context stays the single key-derivation authority. Pure state, no engine calls. Unit-tested (`ArmedStateStoreTest`: product multiplier, summed flats, chained on-hit consumers, per-slot attribution exposure, the payload-free overload, the final-field constructor round-trip, consume-removes, expiry, invulnerability set/expire/clear with an opaque attribution round-trip, clearPlayer wipe).
- New: `cast/RaycastTargeting` - stateless static entity-raycast picking (`pickClosest` single nearest / `pickPiercing` all hits closest-first, capped) over the engine `Selector` + `CollisionMath.intersectRayAABB`, caster excluded by index; `inflateRadius` widens only the broad-phase search. `Hit` carries the ref + ray-t + pick-time position. World-thread.
- New: `cast/BlockRaystep` - stateless static look-ray block walker (`clearDistance` = distance to the first Solid block with optional wall pullback / `hitPosition` = exact hit point); an unloaded / off-map block is treated as clear and stepping continues. All step / distance / pullback values are caller-supplied. World-thread.
- New: `cast/WorldFrameGate` - a once-per-world-frame CAS gate on the millisecond clock (`beginFrame()` is true only for the first call per millisecond). Hold one per world so two worlds ticking in the same millisecond never steal each other's frame. Thread-safe; no world-thread requirement. Unit-tested (`WorldFrameGateTest`).
- New: `cast/WorldEvictors` - the STATIC (JVM-global) world seam: `worldOf(Store|Ref)` resolution + `registerEvictor(Consumer<World>)` + a guarded `onWorldRemoved(World)` fan-out. Eviction is JVM-global so a removed world reaches every consumer's per-world partitions in one place; a consumer's `RemoveWorldEvent` listener calls `onWorldRemoved`.
- New: `cast/WorldKeyedQueues<E>` - a generic per-world queue partition (`queueFor` add path / `peek` nullable drain path / `isEmpty` / `values` / `totalSize` / `evict`) that self-registers a `WorldEvictors` evictor in its constructor. A drain system MUST call `markDrainerAttached()`; a one-time WARNING fires if entries are added before any drainer attaches (entries a no system ticks would leak forever). Each per-world cast ticker swaps its private map for one instance with no tick-body change.
- New: `cast/AbstractWorldFrameSystem` - an abstract `EntityTickingSystem` base (player-query gate) that resolves the world from the store, gates via an internal per-world `WorldFrameGate` (self-evicting), and on winning calls the protected abstract `drainFrame(world, store, commandBuffer)`. NEVER registered by common - the ECS system registry is class-keyed, so each consumer registers its OWN concrete subclass.
- New: `cast/ModelParticleService` - static `ParticleUtil` wrappers: `spawnAt` (position-only, uncapped) + `spawnDirectional` (collects nearby players to reach the rotation-aware overload). A null asset id is a `false` no-op; any error is caught and returns `false` after a guarded FINE log; `true` on success. World-thread. `spawnAt` gained a duration-capped 4-arg overload riding the native `SpawnParticleSystem.maxDuration` field (the same field the first-party `PlayVfxEffect` trigger-volume effect authors via its own `Duration` leaf): a consumer firing a one-shot moment at a bare position can force-stop a particle system authored with an unbounded spawner (a negative `TotalParticles`, meant for a persistent per-entity VFX) instead of it spawning forever there - the fix behind a consuming mod's station completion-flourish leak. Unit-tested (`ModelParticleServiceTest`: the null-asset-id no-op contract on both overloads).
- New: `ui/rows/` - a fixed-slot icon + rich-text LEDGER row primitive, lifted from a consumer's session-summary HUD so any mod's reward / results / summary LEDGER reuses one row model + painter. `SummaryRow` is the immutable value (an icon item id + an already-composed, already-styled client-resolved `Message` + a descriptive semantic `Kind` - `XP`/`CONSUMED`/`PRODUCED`/`LUCKY` generic reward-ledger roles; all colour/bold is baked into the `Message` by the caller, the renderer never branches on `Kind`). `SummaryRowRenderer.render(cmd, rowSelectorPrefix, maxSlots, rows)` paints the list into a FIXED set of pre-declared `.ui` slots the CONSUMER declares (row `i` = `rowSelectorPrefix + i`, each holding an `ItemGrid #Icon` + a `Label #Name`), hides unused / overflow slots, and RETURNS the overflow count for a "+N more" row (`renderWithOverflow` sets that label too). Transport-agnostic (engine `Message` + UI builder only), so any `UICommandBuilder` sink reuses it - a HUD partial update, a page `sendUpdate`, a full `build`. Shipped as a SIBLING to `ui/toast/ToastLine` (same icon-id + `Message` shape, but a transient toast is welded to the toast transport + a `quantity` badge + a per-kind SFX `ToastKind`, whereas a `SummaryRow` is a persistent fixed-slot ledger row) rather than unified, so neither surface forces its semantics onto the other and no third model is forked. The consuming mod keeps only its policy (icon / name resolution, colour conventions, tallies -> rows).
- New: `ui/hud/KeyedCustomHud` - the shared base for a custom in-world HUD overlay, lifted config-free from a consumer's own HUD base so any mod's HUD reads ONE implementation instead of a per-mod copy. Owns a `HudPosition`-driven anchor on a single root element (fixed-height or content-height), two push-throttle strategies (a check-only gate + a CAS atomic acquire), and the native keyed register/lookup/broadcast over Update 5's `HudManager`; a concrete HUD supplies only its identity + layout. Self-contained (no consumer world-lookup utility or logging facade required) - player lookup and "any alive world" broadcast resolution live here, and logging is a locally-guarded call. A consumer with its own HUD base re-parents onto this and keeps only its own conveniences; a consumer with none extends it directly.
- New: `i18n/Msg` - a minimal, mod-agnostic client-resolved `Message` factory, lifted from a consumer's own `i18n.Msg` so several consumer mods can share one arg-binding implementation. Unlike a per-mod `Msg`, `tr(prefix, key, args...)` takes the CALLER's own namespace prefix as an explicit argument (this class carries no fixed namespace); `key`/`raw`/`trNamed`/`keyNamed` round out the translation surface, `join`/`cat` are the same param-safe-composite pair a per-mod `Msg` already carries (`cat` folds through a shipped locale-neutral `common.fmt.cat` glue key so a composite still resolves correctly when nested as another translation's param), and `bold`/`color` are thin fluent style wrappers. The arg-binding invariant (a localized arg is always a nested `Message`, never a pre-resolved String) carries over unchanged.
- New: `entity/PlayerPuppetService` - the "mount the player, hide their body, show a puppet performing the work instead" presentation mechanism, lifted config-free out of a consumer's in-game-proven P0 spike harness. `spawn(accessor, PuppetSpawnRequest)` clones a source player's live `PlayerSkin` (the copy ctor -> `PlayerSkinComponent` -> `CosmeticsModule.createModel` recipe) onto a new networked, `NonSerialized`, `Visible` entity at a caller-resolved position/yaw, with optional held-item mirroring (a bare `InventoryComponent.Hotbar` - the generic "any entity carrying Hotbar + Visible gets networked equipment" mechanism) and an optional `ActiveAnimationComponent` pre-seed on the spawn holder (the render-guaranteed route: a viewer whose tracker marks the puppet newlyVisible on ANY later tick auto-catches-up to the tracked clip, independent of when a direct animation packet happens to land). `PuppetSpawnRequest` is an immutable builder (`sourceRef`/`position`/`yawRadians`/mutually-exclusive `mirrorHeldItem`|`heldItemIdOverride`/`initialAnimation(slot, clipId)`), mirroring `cast.HitResolver.HitRequest`'s established shape. `despawn(puppetRef, Store|CommandBuffer)` (two overloads) and `playAnimation(accessor, puppetRef, slot, itemAnimationsId, clipId, sendToSelf)` (a single-shot primitive; the caller owns its own repeat-beat cadence) round out the spawn surface. Plus the paired SCALE self-hide primitive - `hideByScale(accessor, ref[, nearZeroScale])` returns the prior `EntityScaleComponent` scale as a revert payload for `revealByScale(accessor, ref, priorScale)` (`null` payload removes the component instead of resetting it to `1.0`); a `ModelSwap`-route hide reuses the existing `PlayerModelService.restore` instead of duplicating it. Every spawn-side method takes a `ComponentAccessor<EntityStore>` (the interface both `Store` and `CommandBuffer` implement, the same seam `cast.HitContext` uses) so an interaction-handler-locked caller and a live-`Store` caller both work through ONE method. Plus two pure spatial helpers (`offsetPosition`, `yawRadiansFromDegrees`). WORLD-THREAD ONLY; every engine call is try-guarded to a no-op/null, never a throw. Unit-tested (`PlayerPuppetServiceTest`: the pure helpers, the builder's mutual-exclusivity/default contract, and the try-guard no-throw contract on every engine-touching entry point via a deliberately-null accessor).
- New: `entity/ItemPropEntityService` - spawns/despawns a static, network-replicated, pickup-immune, physics-free PROP entity representing an item at a fixed position (a placed-input display, a shop pedestal, a trophy case, ...), lifted config-free out of a consumer's own placed-input display, itself copied verbatim from the engine's own sanctioned admin "Entity Spawn Page" Items-tab exemplar. Two routes picked by `Item#hasBlockType()`: a block-shaped item spawns a real `BlockEntity` (the actual block model, with the exemplar's own base-scale doubling); everything else spawns a bare `ItemComponent` prop with dropped-item-animation override disabled (the generic "dropped item minus physics" shape); both routes are `PreventPickup`/`PreventItemMerging`/`PropComponent`-marked and `NonSerialized` (never survives a restart). Two-phase API: `buildHolder(accessor, itemId, position, rotation, scale)` returns an un-added `Holder` so a caller can add its own components before committing via `spawn(accessor, holder)`; a one-call `spawn(accessor, itemId, position, rotation, scale)` convenience composes both. `despawn(propRef, Store|CommandBuffer)` mirrors `PlayerPuppetService`'s own shape. WORLD-THREAD ONLY, try-guarded throughout. Unit-tested (`ItemPropEntityServiceTest`: the blank-id short-circuit and the try-guard no-throw contract on every entry point via a deliberately-null accessor).

## 1.3.0
A settings-UI toolkit AND a generic settings-form engine built on it: the reusable binding + write-back + HUD-layout + appendable-row pieces a consumer's in-game config page needs, so a settings page in any mod reuses ONE layer instead of re-deriving it. Plus two per-world-config primitives (a world-name matcher + a generic raw-JSON `Parent`-chain resolver) backing any consumer's "one file per world, inherit from a base" schema. All additive over 1.2.0; a consumer picks up only what it calls.

- New: `world/WorldNameMatcher` - the generic per-world selector: a pre-parsed `Entry<T>(pattern, payload)` list resolved by `resolve(entries, worldName)` with EXACT > longest literal CORE across trailing-`*` PREFIX (`Foo*` = starts with), leading-`*` SUFFIX (`*Foo` = ends with), and leading+trailing CONTAINS (`*Foo*` = contains anywhere) > bare `*` precedence (case-insensitive; on an equal core length the tie-break favours the more anchored form, prefix > suffix > contains; `null` on no match so the caller falls back to its global settings). `*` stays the only metacharacter and only at the ends; `Entry<T>` pre-parses each pattern into a `Kind` + literal core once at load and the public API is unchanged. The CONTAINS form is the one that catches an instance world whose live name carries BOTH a leading `instance-` prefix AND a random suffix (e.g. `instance-KweebecNightmare_Chase-<uuid>`), which a bare trailing-`*` prefix cannot. Dedupes / feature-matches the MMO's `WorldRulesMatcher` shape (the MMO's world rules, the mob-scaling mod's per-world settings). Pure logic, zero engine coupling, unit-tested (`WorldNameMatcherTest`: prefix/suffix/contains, the instance-prefix case, cross-kind longest-core, and the tie-break).
- New: `codec/JsonParentResolver` - generic parent-chain resolution over a keyed RAW-JSON pool, the pre-decode half of native-lite `Parent` inheritance for an asset family whose bodies must merge ACROSS layers the engine store cannot see (jar + pack store bodies PLUS owner-dir files read off disk - the engine's own `Parent` resolution only runs inside one store's load batch). `resolve(pool, outputIds, parentKey, warn)`: per id, deep-clone, read + strip the parent key, resolve the chain transitively (memoized, cycle-guarded; a cycle / unknown parent warns and resolves standalone), child deep-merged OVER parent per leaf via `JsonTreeUtil.deepMergeInto` (arrays replace wholesale); pool ids + parent refs lower-cased; pool-only bases (in the pool, not in `outputIds`) are never emitted. Generalizes the dialogue engine's `DialogueBodyResolver` inherit core (which stays as-is, composing sugar pre-passes + an engine decode). Unit-tested (`JsonParentResolverTest`).
- New: `util/JsonOverrideWriter` - partial-override write-back for a codec-authored owner file (the counterpart to a read/fold config with no `save()`). Sets / removes leaves addressed by DOTTED PascalCase codec paths (`"ZoneHud.Position"`, `"Difficulty.DistanceEscalation.Enabled"`) - preserving every sibling key + a leading `$Comment` - reusing `JsonTreeUtil.deepMergeInto` for the additive merge; PLUS the one thing `deepMergeInto` cannot do, a top-level ARRAY upsert / remove keyed by a match field (whole-element replace / append / remove, other elements preserved - the shape an owner-over-jar concat fold needs). Number type fidelity is the caller's contract (pass an `Integer` for a `Codec.INTEGER` leaf, a `Double` for `Codec.DOUBLE`); a `List`/`String[]` value instead emits a JSON ARRAY leaf (each element boxed the same way), replacing the leaf WHOLESALE like any other; a null value removes a leaf; the public API is plain-Java-typed (String paths, boxed scalars, `Map`/`List`) so a consumer needs no Gson dependency. Atomic write (temp + `ATOMIC_MOVE`), fully guarded (returns `false` on any IO / parse failure, a malformed existing file is NEVER overwritten), unit-tested (`JsonOverrideWriterTest`).
- New: `ui/hud/HudPosition` - the mod-agnostic HUD layout value (a vertical `AnchorEdge` + horizontal `HorizontalEdge` + pixel offsets) LIFTED so every mod's HUD reads one layout authority instead of a per-mod copy. Named corner presets parsed by `parse(preset, offsetX, offsetY)` + `isValidPreset`; `toAnchor(w, h)` (and a content-height `toAnchorContentHeight(w)`) always set `Width` and pin EXACTLY ONE edge per axis so an offset moves the panel instead of being neutralized by an opposing stretch. The `CENTER` horizontal branch pins `Left` only for a nonzero offset (a bare `Horizontal: 0` would stretch full-screen-width, not centre). Carries NO consumer-specific defaults; a consumer supplies its own default position. Unit-tested (`HudPositionTest`).
- New: `ui/SettingsUiUtil` - the domain-free binding + widget helper for a settings / config form (the engine-typed twin of a consumer's own `UIUtil`): dropdown `populate`/`bindDropdown`, `bindTextField`, `bindButton` (1/2/3-arg), `bindNavigation`, plus `setStatus`, `setToggle`, `setTabActive`. Every text-bearing helper targets a Label's `.TextSpans` (via `ZigRichButton`), NEVER a `TextButton.Text` String sink - a client-resolved / parameterized `Message` only substitutes params + renders markup on `.TextSpans` (`setToggle` is a `Button` + inner `#Label` retinted per state via `UiRetint`; `setStatus` sets `.TextSpans`). No i18n keys, no content enums.
- New: `Common/UI/Custom/Pages/ZigListRow.ui` - a generic appendable list row (`#Title` + `#Sub` text column that flexes, an optional `#Badge`, and RICH `#EditBtn` / `#RemoveBtn` action buttons - each a `Button` + inner `#Label` whose label is pushed on `.TextSpans` via `ZigRichButton.text`, so it substitutes / renders markup a `TextButton.Text` cannot; the title is `#Title` so it never collides with the buttons' inner `#Label`) for a CRUD list. Appended into a consumer's list container; the INDEXED PARENT scopes the bare descendant ids, so many rows across many mods never collide (the ZigPartyRow / ZigResultRow rule). For a FIXED form (one instance per control) the reusable unit is the STYLE (`ZigButtons.ui`), not a whole-row doc with fixed inner ids - so a consumer authors its static knobs directly and reuses only the row fragment for the repeated list.
- New: `ui/form/` - the generic settings-form engine. `FieldSpec` describes one field by kind (`TEXT`/`NUMBER`/`CHANCE`/`INT`/`CSV`/`TOGGLE`/`TRISTATE`/`DROPDOWN`/`HEADER`/`NOTE`), an optional dotted-PascalCase leaf path, and an opaque label key the consumer resolves; `SettingsForm` owns the ordered spec list plus a raw value cache and renders itself as appended rows (`buildRows`, wiring a `field`/`press` event convention shared across every text-ish/dropdown/toggle control), refreshes already-built rows for a partial `sendUpdate` (`applyValues`/`applyValue`), and parses the cache back into a leaf map (`collectLeaves(blankIsInherit)`) ready for a `JsonOverrideWriter.setLeaves` write-back, returning a `FormResult` (the leaf map, or the first validation failure's label key + kind). No i18n keys, no consumer content types, and no persistence policy live in the engine; specs, labels, and writing the result stay with the consumer. Any TEXT/NUMBER/CHANCE/INT/CSV/TOGGLE/TRISTATE/DROPDOWN spec may also carry an optional wrapping sub-label via `FieldSpec.withHint(hintKey)` (an immutable copy; throws `IllegalArgumentException` on a `HEADER`/`NOTE` spec, whose row template carries no `#Hint`) for static per-setting help text or a dynamically-composed line (e.g. "Inherits: 30"); `buildRows` paints it, and `SettingsForm.applyHint(cmd, containerSel, fieldId, hint)` sets or hides it independently (a null `Message` hides the row) for a consumer re-pushing a recomposed hint in a partial update. Unit-tested (`SettingsFormTest`).
- New: five appendable `Pages/ZigForm*Row.ui` templates (`ZigFormFieldRow`, `ZigFormDropdownRow`, `ZigFormToggleRow`, `ZigFormHeaderRow`, `ZigFormNoteRow`) backing the `ui/form` engine, following the `ZigListRow.ui` indexed-parent-scoping convention so many rows (and many mods) never collide. The field/dropdown rows reuse the game's own `Common.ui` `@TextField`/`@DropdownBox` components; the toggle row reuses `ZigButtons.ui`'s `@ZigRichRowBtnStyle`. The field/dropdown/toggle templates wrap their existing `#Title` + control in a nested `Group #Row` (fixed height) below which a sibling `Label #Hint` (no fixed height, hidden until painted) can grow a wrapping grey sub-label without disturbing any existing descendant selector.

## 1.2.0
A difficulty-scaling engine primitive plus two ref-less spawn-hook helpers, all additive over 1.1.1, PLUS the dialogue engine's native-inheritance rework. The generic core a consumer scales encounter difficulty on (open-world mob scaling, a future dungeon/raid instance) without re-deriving the fold/clamp math or the pre-add stat plumbing per mod.

- New: `codec/InheritMapCodec` - a generic string-keyed map codec that participates in native asset `Parent` inheritance with per-KEY merge (the stock `MapCodec` whole-replaces on inherit because it is not an `InheritCodec`). On a child overlay it seeds every parent entry, deep-merges each key the child provides when the value codec is itself an `InheritCodec` (e.g. a `BuilderCodec` sub-object), and retains parent-only keys - the opt-in "merging field codec" the framework supports (vanilla `Tags` is the precedent). Reusable for any keyed-map field that should overlay by key under inheritance.
- Change: the branching-dialogue engine moves to NATIVE `Parent` inheritance and the hand-rolled `extends`/`params`/`nodeOverrides`/`extraNodes`/`PruneIfEmpty` template DSL (`dialogue/template/`) is REMOVED. A body may carry a top-level `"Parent": "<id>"`; the new `dialogue/DialogueBodyResolver` resolves the chain (topological, memoized, cycle-guarded) and decodes through the new `DialogueEngine.decodeWithParent` -> the engine's real `decodeAndInheritJson`. `Start`/`Nodes` + node fields are `appendInherited` (omit -> inherit) and the `Nodes` map uses `InheritMapCodec` (child overrides/adds a node BY KEY, inheriting the rest; node fields deep-merge). Option `sugar` stays a pre-pass; loc-keys are convention-derived. `DialogueAssetStore.resolveAll` now runs the resolver over a POOL of templates + bodies (a `DialogueTemplate` is a native `Parent` BASE, not a DSL `extends` target).
- New: dialogue `Type`-condition boolean combinators `AllOf`/`AnyOf`/`Not` (child `Conditions` list under `All`/`Any`/`Of`), and a `DialogueNode` may self-gate with its own `Conditions` (`resolveEntryNodeId` requires them too) - so state-varied beats collapse onto one node map with runtime conditions instead of `(node x state)` + Start-row duplication.
- New: a `DialogueOption` `Presentation` (nested `{Color, Icon:{Item,Glyph}}` codec) overrides the action-derived `DialogueOptionStyle`: a per-option button colour (hover/press derived from it) and an icon by game-item id (rendered via the new `#OptIconItem` grid slot in `ZigDialogueOptionRow.ui`) or a fixed glyph token.
- New: `DialogueOptionTheme` - the option-style DEFAULTS (colour + glyph per kind) are now data-driven, not `DialogueOptionStyle` enum literals. A Pattern-A `dialogue/asset/DialogueOptionThemeAsset` (structured codec, PascalCase `Color`/`HoverColor`/`PressColor`/`Glyph`, NO raw `Payload`) folds `defaults < pack < owner` through `dialogue/DialogueOptionThemeConfig` keyed by `DialogueOptionStyle.key()` (`accept`/`turnin`/`continue`/`neutral`/`farewell`); common ships the neutral defaults as its OWN asset pack (`Server/ZiggfreedCommon/DialogueOptionTheme/*.json`). `DialoguePage.applyOptionLook` resolves each colour/glyph leaf from the theme (a missing hover/press derives from `Color`), a per-option `Presentation` still overrides, and the enum stays ONLY as the per-leaf fail-closed fallback. A consumer pack or server owner retints every dialogue option by dropping a same-id file, no Java. Glyph TEXTURES stay in `ZigDialogueOptionRow.ui` (a Java-set texture path renders a red X); the theme drives the colour + which glyph shows. Guarded by `AssetCodecInitTest`.
- New: a `DialogueOption` `Style` field - an option can declare its themed look BY KIND (`"Style": "accept"`/`"turnin"`/`"continue"`/`"neutral"`/`"farewell"`), which `DialogueEngine.classifyOption` resolves through `DialogueOptionStyle.byKey` to OVERRIDE the action-derived style. The colour + glyph then come from the data-driven `DialogueOptionTheme` for that kind - a theme-resolved, one-edit-changes-everywhere alternative to hard-coding a per-option `Presentation` colour/glyph (e.g. a page-opening option that should read as a green "accept" affordance).
- New: shared OPTION FRAGMENTS - the intra-dialogue DRY seam. A body may declare a top-level `"Fragments": { "<id>": [ <option>, ... ] }` map of reusable option lists, and a node references one or more with `"IncludeOptions": ["<id>", ...]`; `DialogueBodyResolver` DEEP-COPIES and appends each fragment's options after the node's own `Options` (pre-sugar, pre-decode), then strips `Fragments` + every `IncludeOptions` before the codec sees them. Authored ONCE, a shared footer (hub `Got any quests?` / menu / return-to-npc options) is spliced into every steady node instead of inlined N times; an unknown fragment id warns and is skipped. Fragments resolve within the SAME body (not visible across a `Parent`). No-op for any dialogue that declares neither, so existing dialogues are unaffected.

- New: `scaling/` - the generic, domain-free difficulty-scaling engine. `ScalingContext` (the one input record: `baseDifficulty` + a `double[] participantPowers` + an `AggregationMode` + an opaque `@Nullable Object instanceHandle`) serves BOTH open-world proximity groups and 1-10 player instances with zero consumer-domain types (no player/party/skill leaks in); a consumer reduces each participant to a `double` at the call site. `AggregationMode` folds the array (`SOLO`/`AVERAGE`/`PEAK`/`WEIGHTED`/`DISABLED`, plus a lenient `fromName` string parse). `PowerAggregation.fold` is the pure fold (empty/DISABLED -> `0.0` identity; `WEIGHTED` = `sum(p^2)/sum(p)`, high powers drive). `ScalingEngine.resolve(ctx, bandWidth, minCap, maxCap)` = `clamp(base + clamp(aggregatedPower - base, -bandWidth, +bandWidth), minCap, maxCap)`, returning the base untouched for a DISABLED/empty context. PURE logic, zero engine coupling, unit-tested (`ScalingEngineTest`/`PowerAggregationTest`). A consumer builds the context (open-world: a cached region power scalar via `ScalingContext.openWorld`; instanced: a party reduced to a `double[]` with a non-null handle) and calls the SAME engine for both legs.
- New: `health/HealthUtil.scaleMaxHealth(Holder, factor, key)` - the ref-less overload of the existing `scaleMaxHealth`, raising a mob's `Health` MAX (a multiplicative modifier keyed for idempotency) and healing to the new max, reading the `EntityStatMap` straight off a pre-add `Holder`. The seam for scaling HP INSIDE a `HolderSystem.onEntityAdd` spawn hook, before the entity has a valid `Ref`. Idempotent per holder (no-op when the key modifier is present, the stat is not yet balanced, or the factor is 1.0); world-thread, fully try-guarded.
- New: `util/EntityIdentifierUtil.roleName`/`roleIndex` (Store/Ref AND `Holder` forms) - read an NPC's `NPCEntity` role identity: `roleName` (the restart-STABLE string to key allow/deny/classification config on) and `roleIndex` (the restart-UNSTABLE integer for hot in-tick lookups only). `null`/`-1` for a non-NPC or on any error. The `Holder` forms let a pre-add spawn hook classify a mob before it has a ref.
- New: `health/HealthUtil.reconcileMaxHealth(Holder, factor, key)` + a `Store`/`Ref` form - the RECONCILE counterpart to `scaleMaxHealth`. Where `scaleMaxHealth` is add-only (no-op when the key is present), `reconcileMaxHealth` CONVERGES the stored MAX modifier to the current factor: absent + factor!=1 -> put + maximize (first-apply heal); present + factor==1 -> remove; present + a different amount -> replace WITHOUT maximize (a shrink auto-clamps via the engine recalculate; a re-maximize would full-heal on every retune). The seam a persist-and-re-derive spawn hook calls unconditionally so a re-derived scale on chunk reload never strands a stale HP value; world-thread, fully try-guarded. `scaleMaxHealth` semantics are UNCHANGED (Kweebec's `BossController` relies on its idempotent no-op-when-present contract).
- New: `instance/effect/EntityEffectService.apply(ref, effectId, CommandBuffer)` + a `Store` form - an ASSET-AUTHORITATIVE apply that uses the effect asset's own `Duration` + `OverlapBehavior` + `Infinite` flag (no Java-side duration/overlap constants), via the engine's index-aware `addEffect` which auto-dispatches to the infinite path on `Infinite:true`. One call serves both an infinite aura/stat effect and a duration-authored on-hit effect, and a mis-authored `Infinite` flag can never pin a timed effect forever. Lets a consumer delete Java `DURATION`/`OVERWRITE` constants and make the effect asset the single schema authority.
- New: `util/SplitMix64` - a mod-agnostic deterministic SplitMix64 seeded stream (`nextLong`/`nextDouble` + the static `mix(a,b)` two-long fold) for any path needing REPRODUCIBLE randomness (a chunk-reload-stable roll, a cross-server-stable rotation) that cannot use `java.util.Random`. Lifted verbatim from the MMO mob-scaling mod (its local copy is deleted). Do NOT repoint an already-shipped stream at `mix` unless it was bit-identical.
