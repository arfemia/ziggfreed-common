# CLAUDE.md - Ziggfreed Common

A **standalone Hytale utility mod** that holds shared, mod-agnostic primitives lifted (config-free) from the MMO Skill Tree mod and the Kweebec Nightmare minigame, so standalone Ziggfreed minigames AND the MMO Skill Tree consume ONE battle-tested implementation instead of re-deriving 3D sound / camera / HUD plumbing per mod (hyMMO 1.4.0 consumes the dialogue engine, toast engine, JsonTreeUtil, AbstractRawJsonAsset, the `ui/UiRetint` retint engine + the `ui/theme` Palette/ThemeRecord model, and the `AbstractKeyedAssetConfig` fold as a HARD runtime dependency; Kweebec Nightmare 1.0.0 consumes the full primitive set). **Status: v1.4.0** (1.4.0 adds the reusable cast / ability RUNTIME layer [`cast/`](src/main/java/com/ziggfreed/common/cast/CLAUDE.md) - a per-consumer on-hit builder registry + armed-state store, ray/block targeting, and the per-world tick partition (frame gate, world-keyed queues, evictor fan-out, an abstract drain system) plus a guarded particle seam - lifted config-free for a consumer that fires abilities, ships EMPTY of content; the MMO is the first consumer; 1.3.0 adds a settings-UI toolkit AND a generic settings-FORM engine built on it - `util/JsonOverrideWriter` (partial owner-file write-back over `JsonTreeUtil.deepMergeInto` + an array-by-key upsert, plus array-leaf write-back: a `List`/`String[]` value emits a JSON array, replacing wholesale), `ui/hud/HudPosition` (the lifted mod-agnostic HUD layout value), `ui/SettingsUiUtil` (a domain-free form-binding helper), the `Pages/ZigListRow.ui` CRUD row, and `ui/form/` (an ordered `FieldSpec` list rendered/refreshed/collected by `SettingsForm` into appendable rows, ready for a `JsonOverrideWriter` write-back, plus its five `Pages/ZigForm*Row.ui` templates, each with an optional per-row wrapping hint line) - the reusable pieces an in-game config page needs - PLUS two per-world-config primitives: `world/WorldNameMatcher` (the generic exact > longest-core prefix/suffix/contains > bare `*` per-world selector, feature-matched to the MMO's `WorldRulesMatcher` so a `*Foo*` contains Match catches an `instance-`-prefixed, randomly-suffixed instance world) and `codec/JsonParentResolver` (generic raw-JSON `Parent`-chain resolution across store + owner-dir layers), backing a consumer's "one file per world, inherit from a base" schema (the mob-scaling mod's `Worlds/*.json` is the first consumer); 1.2.0 added the domain-free `scaling/` difficulty-scaling engine + a ref-less `HealthUtil.scaleMaxHealth(Holder,...)` overload + `EntityIdentifierUtil` `roleName`/`roleIndex` NPC role reads, the Phase-0 lift the MMO mob-scaling mod builds on; 1.1.0 added the `ui/UiRetint` retint engine + `ui/theme` value model + `world/AtmosphereService`/`ForcedMusicService`; see CHANGELOG) - the static primitives PLUS a generic branching NPC **dialogue engine** ([`dialogue/`](src/main/java/com/ziggfreed/common/dialogue/CLAUDE.md)) PLUS the **matchmaking lobby** ([`lobby/`](src/main/java/com/ziggfreed/common/lobby/CLAUDE.md)) and the **instance-experience layer** (a full party system, queue/end-game/leaderboard SCREENS, a codec-asset-driven instance preset + reward model with a full-inventory guard - see below); it SHIPS its `.ui` asset pack (`IncludesAssetPack:true`) and, as of the asset-driven-framework pass, the plugin now REGISTERS the framework asset stores (`setup()` calls [`asset/FrameworkAssetRegistrar`](src/main/java/com/ziggfreed/common/asset/FrameworkAssetRegistrar.java)) at `Server/ZiggfreedCommon/<Type>/` - see the asset-driven paradigm below.

> **The instance-experience layer (the reusable end-game + party/queue framework).** A consumer minigame (Kweebec now, future dungeons/raids) gets, mod-agnostically: a searchable-invite **party** ([`party/`](src/main/java/com/ziggfreed/common/party/) - `Party`/`PartyService` + `PartyInvitePage`, the Queue button carrying a chosen preset), a unified **Play / queue-mode** screen ([`instance/play/`](src/main/java/com/ziggfreed/common/instance/play/CLAUDE.md) - `PlayModePage`: an asset-driven Public/Party/Solo chooser that morphs to a still-closable live roster whose launch timer ticks down on screen; atomic party group-join + immediate solo-launch in `lobby/`), an **end-game results** screen ([`instance/result/`](src/main/java/com/ziggfreed/common/instance/result/) - `MatchResult` team breakdown + reward chips + leaderboard CTA via `ResultsPage`), a generalized bucketed **leaderboard** ([`instance/leaderboard/`](src/main/java/com/ziggfreed/common/instance/leaderboard/) - `Leaderboard` + `LeaderboardPage`, lifted from Kweebec), an asset-driven **reward** model with the no-claim-with-full-inventory guard PLUS score-tiered **loot tables** ([`instance/reward/`](src/main/java/com/ziggfreed/common/instance/reward/CLAUDE.md) - `InstanceReward`/`InstanceRewardGranter`/`PendingRewardStore`, `InventoryUtil.canFit`; `LootTable`/`LootEntry`/`LootTableAsset` = weighted, score-gated, quantity-ranged pools a preset references by `RewardTableId` so a higher run score unlocks premium entries + more rolls), a codec-asset-driven **preset** ([`instance/preset/`](src/main/java/com/ziggfreed/common/instance/preset/) - `InstancePresetAsset` Pattern A + `InstancePresetConfig`), and the **generic Pattern-A registrar lift** ([`asset/`](src/main/java/com/ziggfreed/common/asset/) - `AssetStoreRegistrar`/`AssetMergeAdapter`). Each page takes an immutable `*PageDeps` + a locale-free `*Messages`/`*ScreenMessages` provider (the consumer supplies pre-built client-resolved `Message`s); pages reuse the shared `ZigFrames.ui`/`ZigButtons.ui` styles AND the lifted `ui/toast/` engine for in-page reward/feedback toasts (results reward strip, party invite/privacy actions, queue-joined). Reward chips render an item icon by id + an auto-generated "x{n} {ItemName}" label from the item's own engine display name (`RewardChipRenderer`; no per-reward `displayKey` needed). The party screen carries an owner public/private queue toggle. Kweebec's consumer wiring is its `experience/` package - the model for any future consumer.

> **The reusable ENCOUNTER / difficulty framework (Kweebec today; the MMO's future dungeons/raids are the target).** Beyond the end-game layer, a consumer gets mod-agnostic primitives to BUILD an encounter: per-entity timed effects + escalation bands ([`instance/effect/`](src/main/java/com/ziggfreed/common/instance/effect/CLAUDE.md) - `EntityEffectService.applyTimed`/`applyBand` + `EffectBand`/`EffectBandLadder`, the on-hit slow / debuff / pace-band seam over `EffectControllerComponent`, lifting Kweebec's hunter-pace swap); weighted spawn rosters + a wave director ([`instance/encounter/`](src/main/java/com/ziggfreed/common/instance/encounter/CLAUDE.md) - `SpawnRoster<T>`/`SpawnUnit<T>` weighted + tier-gated picks, `EncounterDirector` cooldown/cap/per-round-max gating; PURE logic, no engine deps); floor-snapped runtime placement ([`world/SpawnPlacement`](src/main/java/com/ziggfreed/common/world/CLAUDE.md) - ring/near-player + foliage-skip surface snap over `SurfaceProbe`); a co-op zone-hold objective timer ([`instance/zone/`](src/main/java/com/ziggfreed/common/instance/zone/CLAUDE.md) - `ZoneHoldTimer`, the "required group holds a zone TOGETHER for X seconds, reset on break" state machine, for extraction pads / capture points / king-of-the-hill; PURE logic); a damage-cause index cache ([`util/DamageCauseCache`](src/main/java/com/ziggfreed/common/util/CLAUDE.md)); and a `RoundMetadata` outbound-event envelope ([`instance/metadata/`](src/main/java/com/ziggfreed/common/instance/metadata/CLAUDE.md)). **Kweebec Nightmare 0.5.0 is the exemplar consumer** (hunter on-hit slow + bonus damage + proximity-stacking + enrage; asset-driven spawn triggers near players on shrine-light + at corruption milestones; a multi-phase reskinned boss) - the SAME primitives back any future MMO dungeon/raid. Design + verified engine seams + in-game spikes: [[kweebec-difficulty-spawn-design]].

It has **zero MMO dependency and NO Perfect Utils coupling** - the only dependency is the Hytale server jar. The plugin registers ziggfreed-common's own framework asset stores (`FrameworkAssetRegistrar.registerAll` in `setup()`) and nothing else; the stateless static primitives register nothing - a consumer calls them directly.

> **PARADIGM (remember this): a generic, mod-agnostic Hytale primitive belongs HERE, not duplicated in a consumer mod.** When a sibling mod (Kweebec Nightmare, a future minigame, the MMO) needs a reusable engine wrapper - 3D sound, camera, an asset-index resolve, a HUD/toast helper, a worldgen/terrain query, etc. - add it to `ziggfreed-common` and consume it; do NOT reimplement it per mod. New primitive = a new file here (+ its package router), then rebuild the jar consumers pin. A consumer keeping only the mod-SPECIFIC policy (round logic, content config) is correct; the reusable engine call underneath is what gets lifted. (Latest example: `world/SurfaceProbe`, lifted out of Kweebec's `ArenaBuilder` so any mod can floor-snap onto procedural terrain.)

> **PARADIGM (the goal is no-Java pack authoring): consumer CONTENT + display CONFIG belongs in `/Server/` pack JSON, NEVER Java-baked.** Anything a pack author should be able to create or tune without code - dialogues, the queue-mode cards, leaderboard tabs/columns, party knobs, reward lists, queue policy - is a codec asset loaded from `Server/<Mod>/<Type>/` (Pattern A structured codec, or a raw-`Payload` type resolved by native `Parent` inheritance - dialogues - or an `extends`/`params` DSL), resolved `defaults < pack < owner`. The library ships at most a NEUTRAL structural fallback (e.g. `QueueModeSet.fallback()` has no baked icons) and ZERO content; the real values are authored in `/Server/ZiggfreedCommon/<Type>/`. A Java constant list / hardcoded JSON string that represents content is a smell - lift it to an asset. **Common OWNS its framework stores** (one class = one registrant; a consumer must NOT re-register them - it authors JSON into the path and reads the resolved config singleton back). The registered framework asset types ([`asset/FrameworkAssetRegistrar`](src/main/java/com/ziggfreed/common/asset/FrameworkAssetRegistrar.java), all via the [`asset/AbstractKeyedAssetConfig`](src/main/java/com/ziggfreed/common/asset/AbstractKeyedAssetConfig.java) fold + the Pattern-B [`asset/AbstractRawJsonAsset`](src/main/java/com/ziggfreed/common/asset/AbstractRawJsonAsset.java) base): **Dialogues**/**DialogueTemplates** ([`dialogue/asset/`](src/main/java/com/ziggfreed/common/dialogue/asset/), native `Parent` inheritance via per-consumer-engine `resolveAll` -> `DialogueBodyResolver`; a `DialogueTemplate` is a `Parent` base), **Instances** ([`instance/preset/InstancePresetAsset`](src/main/java/com/ziggfreed/common/instance/preset/InstancePresetAsset.java)), **LootTables** (instance/reward/LootTableAsset), **Bosses** (`instance/encounter/MultiPhaseBossAsset`), **BandedEffects** (`instance/effect/BandedEffectAsset` -> `EffectBandLadder`), **EncounterRules** (`instance/encounter/EncounterRuleAsset`, free-string Trigger), **PrefabPlacements** (`world/WeightedPrefabPlacementAsset`), **Leaderboard** (`instance/leaderboard/LeaderboardLayoutAsset`), **Arenas** (instance/arena/ArenaDefinitionAsset), **Party** (`party/PartySettingsAsset`). A consumer reading any of these must do so LAZILY / post-load (the configs are populated by `LoadedAssetsEvent` AFTER plugin `setup()`). Plan + remaining consumption work: [[ziggfreed-common-asset-driven-configs]].

## Build

Gradle runs via PowerShell (Java 25). Self-contained `build.ps1` builds + installs:
```powershell
cd 'D:\dev\business\hyMMO\additional-mods\ziggfreed-common'; .\build.ps1
.\build.ps1 -Install:$false     # build only
.\build.ps1 -ModsDir <path>     # explicit install target (else $env:HYTALE_MODS_DIR)
```
Produces `build/libs/ZiggfreedCommon-<version>.jar` and copies the runtime jar (never `-sources`/`-javadoc`) into the Hytale `Mods/` folder when `HYTALE_MODS_DIR` is set. `.\gradlew.bat build` works too. The Hytale server jar is referenced `compileOnly` (path in `gradle.properties`: `hytaleHome`/`patchline`/`game_build`) and NEVER bundled; jsr305 is `implementation` (the `@Nonnull`/`@Nullable` annotations must resolve for consumers that compile against this jar).

## Consuming this mod

A consumer mod adds ziggfreed-common as a `compileOnly files(...)` dependency (the local jar path until a remote + submodule gitlink exist) and lists it in its `manifest.json` `Dependencies` (or `OptionalDependencies`) so the server loads it first. NEVER bundle this jar into a consumer (double-loading engine-touching classes under two classloaders breaks identity). All primitives are static, config-free, and reusable as-is.

## Architecture (per-package routers carry the working detail)

Each domain package carries a nested `CLAUDE.md` router that loads when you touch that subtree. One line each:

- **[`sound/`](src/main/java/com/ziggfreed/common/sound/CLAUDE.md)** - `Sound3D`: play a `SoundEvent` by id at a position OR an entity ref, with a `SoundCategory` + an optional per-listener `Predicate<Ref>`; index resolution through `AssetIndexCache`. World-thread.
- **[`camera/`](src/main/java/com/ziggfreed/common/camera/CLAUDE.md)** - `CameraShakeService` (one-shot `CameraEffect` shake) + `ServerCameraService` (`SetServerCamera` top-down apply/reset). Packet-only; thread-safe writes.
- **[`util/`](src/main/java/com/ziggfreed/common/util/CLAUDE.md)** - `AssetIndexCache` (the "cache ONLY a positive index" resolver), `NumberFormatter`, `CommandExecutor` (execute-only), `HostilityUtil`, `EntityIdentifierUtil`, `DamageCauseCache` (memoized `DamageCause` index, for identifying a custom damage cause in a `DamageEventSystem`).
- **[`inventory/`](src/main/java/com/ziggfreed/common/inventory/CLAUDE.md)** - `InventoryUtil` (give/count/take/spend a custom resource item by id across the combined inventory sections; world-thread, try-guarded).
- **[`health/`](src/main/java/com/ziggfreed/common/health/CLAUDE.md)** - `HealthUtil` (native `EntityStats` heal: `fullHeal` / `heal(amount)` / `fullRestore`, via `EntityStatMap.addStatValue` + `DefaultEntityStatTypes` - the same API the MMO health-tick path uses; world-thread, try-guarded). Plus `scaleMaxHealth` in a `Store`/`Ref` form AND a ref-less `Holder` form (raise an entity's Health MAX inside a pre-add `onEntityAdd` spawn hook).
- **[`scaling/`](src/main/java/com/ziggfreed/common/scaling/CLAUDE.md)** - the generic, domain-free difficulty-scaling engine: `ScalingContext` (base difficulty + a `double[]` of participant powers + `AggregationMode` + an opaque instance handle) folded by `PowerAggregation` and resolved by `ScalingEngine.resolve(ctx, bandWidth, minCap, maxCap)` into a band-clamped effective difficulty. Pure logic, zero engine coupling; the ONE engine open-world mob scaling and future instance scaling both call (only the context inputs differ). The Phase-0 lift for the MMO mob-scaling mod.
- **`feedback/`** - `Notify` (Default/Danger/Warning/Success toasts) + `EventTitles` (centered banner). Both take a pre-built `Message`.
- **[`ui/`](src/main/java/com/ziggfreed/common/ui/CustomHudHelper.java)** - `CustomHudHelper` (install a consumer-built `CustomUIHud` + strip/restore the native HUD) + **[`ui/ZigRichButton`](src/main/java/com/ziggfreed/common/ui/ZigRichButton.java)** (the CLICKABLE-rich-text primitive: a vanilla `Button` element wrapping a `Label #Label`, so parameterized / glue / `Msg.join` text substitutes on `.TextSpans` - what a `TextButton.Text` cannot do; `text`/`color`/`iconStart`/`iconEnd` helpers, the shared tintable `@ZigRichRowBtnStyle` + `@ZigRichPrimaryBtnStyle` + reusable `@ZigRichBtn` component in `ZigButtons.ui`) + **[`ui/UiRetint`](src/main/java/com/ziggfreed/common/ui/UiRetint.java)** (the generic palette-to-selector retint primitive: `retintColor`/`retintButtonStates`/`retintStyleStates`/`swapPatch`/`isSixDigitHex` - resolved hex + selector params only, no theme model / gate / policy; `ToastRenderer`/`DialoguePage`/`LeaderboardPage`/`PartyInvitePage` converge onto it, and the MMO's `ThemeRetint` is a thin policy wrapper over it) + **[`ui/theme/`](src/main/java/com/ziggfreed/common/ui/theme/Palette.java)** (the mod-agnostic theme VALUE MODEL `Palette` + `ThemeRecord` that pairs with `UiRetint`: pure colour-slot + optional-texture data + a neutral `premium` tag, foldable over `AbstractKeyedAssetConfig`; license-free - the consumer owns the gate. Lifted from the MMO's `UIThemeConfig`), plus **[`ui/toast/`](src/main/java/com/ziggfreed/common/ui/toast/CLAUDE.md)** - the lifted transport-agnostic in-page toast engine (`ToastController`/`ToastRenderer`/`ToastSpec`/`ToastLine`/`ToastKind` + a `ToastablePage<T>` page base, shipping its own neutral `Pages/ZigToast.ui`). i18n is parameterized OUT (pre-built `Message`s only). The hyMMO MMO is the shared-file owner and re-derives its own `ToastablePage` over this engine. Plus **`ui/SettingsUiUtil`** (the domain-free settings-form binding helper: `populate`/`bindDropdown`/`bindTextField`/`bindButton`/`bindNavigation` + `setStatus`/`setToggle`/`setTabActive`, all text on `.TextSpans` via `ZigRichButton`, never a `TextButton.Text` sink), **[`ui/form/`](src/main/java/com/ziggfreed/common/ui/form/CLAUDE.md)** (the generic settings-FORM engine built on top: `FieldSpec` + `SettingsForm` render/refresh/collect appendable rows from one ordered field-spec list, plus five `Pages/ZigForm*Row.ui` templates - see its router), and **[`ui/hud/HudPosition`](src/main/java/com/ziggfreed/common/ui/hud/HudPosition.java)** (the lifted mod-agnostic HUD layout value: corner presets via `parse`/`isValidPreset` + `toAnchor(w,h)`/`toAnchorContentHeight(w)` pinning exactly one edge per axis; no consumer-specific defaults). The reusable `Pages/ZigListRow.ui` CRUD row (title/subtitle + optional badge + Edit/Remove) pairs with them for a list editor.
- **[`world/`](src/main/java/com/ziggfreed/common/world/CLAUDE.md)** - `SurfaceProbe` (top-solid-Y column probe via `World.getBlock`, skipping transparent foliage; world-thread, try-guarded) to floor-snap runtime-placed prefabs/entities onto procedural terrain; `SpawnPlacement` (ring-around / near-player runtime spawn positions, each surface-snapped with a foliage-skip set so a runtime spawn lands on GROUND, not canopy).
- **[`cast/`](src/main/java/com/ziggfreed/common/cast/CLAUDE.md)** - the reusable cast / ability RUNTIME layer: `OnHitRegistry` (per-consumer on-hit builder dispatch) + `CastParams` (lenient param coercion) + `ArmedStateStore` (per-caster next-hit / invulnerability state, caller-supplied UUID key) + `RaycastTargeting` / `BlockRaystep` (entity + block look-ray targeting) + the per-world tick partition (`WorldFrameGate` millisecond CAS gate, `WorldEvictors` JVM-global world-resolve + eviction fan-out, `WorldKeyedQueues<E>` self-evicting per-world queue partition, `AbstractWorldFrameSystem` the abstract per-world drain base) + `ModelParticleService` (guarded `ParticleUtil` spawns). Ships EMPTY of content; a consumer registers its own on-hit types and its own concrete drain subclass. World-thread where it touches the store; pure/thread-safe otherwise.
- **[`instance/effect/`](src/main/java/com/ziggfreed/common/instance/effect/CLAUDE.md) + [`encounter/`](src/main/java/com/ziggfreed/common/instance/encounter/CLAUDE.md) + [`metadata/`](src/main/java/com/ziggfreed/common/instance/metadata/CLAUDE.md)** - the reusable ENCOUNTER framework (see the blockquote above): `EntityEffectService`/`EffectBand`/`EffectBandLadder` (timed + banded per-entity effects over `EffectControllerComponent`), `SpawnRoster<T>`/`SpawnUnit<T>`/`EncounterDirector` (weighted tier-gated spawn picks + cooldown/cap/per-round-max wave gating, pure logic), `RoundMetadata` (outbound integration envelope). Kweebec 0.5.0 is the exemplar; the MMO's dungeons/raids are the target.
- **[`instance/zone/`](src/main/java/com/ziggfreed/common/instance/zone/CLAUDE.md)** - `ZoneHoldTimer`: a continuous "the required group must hold this zone TOGETHER for X seconds, RESET on break" objective timer (pure logic, `nowMillis` injected like `EncounterDirector`). `update(occupants, required, now)` latches complete; `isHolding()`/`remainingSeconds(now)`/`progress(now)` drive a HUD. The consumer owns WHO/WHERE/what-on-complete. Exemplar: Kweebec's Chase extraction platform.
- **[`worldmap/`](src/main/java/com/ziggfreed/common/worldmap/CLAUDE.md)** - `WorldMapMarkers`: place/remove world-map + compass POI markers over the native `WorldMapManager` - global (all players, via the engine `POIMarkerProvider`) AND per-player (`registerProvider` returning each player's own markers). Build a marker with `marker(id, icon, x,y,z, name)`. Needs the world's compass/map enabled to render. `MapDiscovery` (+ `DiscoveryMode`) builds on it: a hidden-until-discovered POI tracker with two orthogonal axes - trigger (`OFF`/`ON_INTERACT` via `discover` / `PROXIMITY` via the pure `revealWithin`) x per-POI visibility (`PER_PLAYER`/`SHARED`); `attach(world)` enables the compass + registers one provider.
- **[`dialogue/`](src/main/java/com/ziggfreed/common/dialogue/CLAUDE.md)** - the generic branching NPC dialogue engine: a per-consumer `DialogueEngine` (builder pre-seeds generic `Goto`/`Close`/`SetFlag`/`Talk`/`OpenPage` actions + `Flag`/`NotFlag` `Type`-list conditions, the consumer registers domain types + seams), the executor/sugar/native-`Parent`-resolver (`DialogueBodyResolver` + `InheritMapCodec`, no template DSL)/structural-validator, and a `page/DialoguePage` whose `.ui` ships in this jar. The hyMMO MMO consumes + extends it; a minigame builds with generics only. NOT a static-util like the others - it is built per consumer.
- **[`codec/`](src/main/java/com/ziggfreed/common/codec/CLAUDE.md)** - inheritance-aware codec + raw-JSON merge primitives: `InheritMapCodec` (per-key map merge under native `Parent` inheritance) + `JsonParentResolver` (generic parent-chain resolution over a keyed raw-JSON pool, for asset families merging across store + owner-dir layers).
- **[`asset/`](src/main/java/com/ziggfreed/common/asset/CLAUDE.md)** - the generic Pattern-A registrar lift + the framework asset-store plumbing (`FrameworkAssetRegistrar`, `AbstractKeyedAssetConfig`, `AbstractRawJsonAsset`, `AssetStoreRegistrar`/`AssetMergeAdapter`); Common OWNS these stores, a consumer authors JSON + reads the resolved config back.
- **[`entity/`](src/main/java/com/ziggfreed/common/entity/CLAUDE.md)** - `PlayerModelService` (swap a player's visible model).

## Conventions

`@Nonnull`/`@Nullable` on params; `ZiggfreedCommonPlugin.LOGGER` for logging (guard the raw LOGGER behind a try/catch on any path a unit test could reach - the flogger LOGGER throws in a log-manager-less unit JVM). `ConcurrentHashMap` for shared maps. Every engine-touching call is try-guarded so a missing asset / bad ref degrades to a no-op, never a throw into the caller. World-thread discipline: off-thread work hops via `world.execute` before any Store/Ref/Sound/HUD/camera read; packet writes (`writeNoCache`/`write`) are thread-safe off-thread. **No em-dashes anywhere.** Package root `com.ziggfreed.common`. **UI RICH TEXT (any primitive that shows text): colour/bold/italic = native `<color is="#hex">`/`<b>`/`<i>`/`\n` markup in the `.lang`, rendered on a Label's `TextSpans` (NEVER `.Text`, which prints the tags literally); per-locale, client-parsed; `TextButton`s + String sinks can't (NOR do they substitute `{0}/{1}` params - a parameterized `Msg.tr` prints the literal `"{0}{1}"` on a `TextButton.Text`). For CLICKABLE rich / parameterized text use [`ui/ZigRichButton`](src/main/java/com/ziggfreed/common/ui/ZigRichButton.java): the vanilla `Button` ELEMENT (a `ButtonStyle` container) wrapping a child `Label #Label` set via `.TextSpans`, styled by `ZigButtons.ui` `@ZigRichRowBtnStyle` (theme-tintable). The `dialogue/` page does the Label case (`#NodeText.TextSpans`). Crucial UX infra, authority: `../../.claude/research/hytale-rich-text-textspans.md`.**
- **HARD RULE - every LABELED clickable button is a `ZigRichButton` (a `Button` + inner `#Label`), NEVER a `TextButton` whose `.Text` you set from Java.** We HAVE the rich-button primitive - use it. A `Button` + `#Label` pushed via `ZigRichButton.text` (`.TextSpans`) substitutes params + renders markup + resolves cleanly; a `TextButton.Text` sink is the wrong tool (it is the exact anti-pattern this primitive exists to replace). This applies to page tabs, save / confirm / cancel buttons, list-row action buttons - anything with a label. `SettingsUiUtil.setToggle` / `setTabActive` already drive the rich `#Label`; a page authors its buttons as `Button { Style: $Z.@ZigRichRowBtnStyle; Label #Label {...} }` (or `$Z.@ZigRichBtn`) and sets each label with `ZigRichButton.text`. Do NOT reach for a `TextButton` label just because it is a single key.

**Submodule order (when a remote exists):** commit + push HERE first, verify the SHA is on the remote, THEN bump the gitlink in the parent hyMMO repo. Until a remote exists, consumers build against the local jar.
