# CLAUDE.md - Ziggfreed Common

A **standalone Hytale utility mod** that holds shared, mod-agnostic primitives lifted (config-free) from the MMO Skill Tree mod and the Kweebec Nightmare minigame, so standalone Ziggfreed minigames (and eventually the MMO) consume ONE battle-tested implementation instead of re-deriving 3D sound / camera / HUD plumbing per mod. **Status: v0.3.x** - the static primitives PLUS a generic branching NPC **dialogue engine** ([`dialogue/`](src/main/java/com/ziggfreed/common/dialogue/CLAUDE.md)) PLUS the **matchmaking lobby** ([`lobby/`](src/main/java/com/ziggfreed/common/lobby/CLAUDE.md)) and the **instance-experience layer** (a full party system, queue/end-game/leaderboard SCREENS, a codec-asset-driven instance preset + reward model with a full-inventory guard - see below); it SHIPS its `.ui` asset pack (`IncludesAssetPack:true`) and, as of the asset-driven-framework pass, the plugin now REGISTERS the framework asset stores (`setup()` calls [`asset/FrameworkAssetRegistrar`](src/main/java/com/ziggfreed/common/asset/FrameworkAssetRegistrar.java)) at `Server/ZiggfreedCommon/<Type>/` - see the asset-driven paradigm below.

> **The instance-experience layer (the reusable end-game + party/queue framework).** A consumer minigame (Kweebec now, future dungeons/raids) gets, mod-agnostically: a searchable-invite **party** ([`party/`](src/main/java/com/ziggfreed/common/party/) - `Party`/`PartyService` + `PartyInvitePage`, the Queue button carrying a chosen preset), a unified **Play / queue-mode** screen ([`instance/play/`](src/main/java/com/ziggfreed/common/instance/play/CLAUDE.md) - `PlayModePage`: an asset-driven Public/Party/Solo chooser that morphs to a still-closable live roster whose launch timer ticks down on screen; atomic party group-join + immediate solo-launch in `lobby/`), an **end-game results** screen ([`instance/result/`](src/main/java/com/ziggfreed/common/instance/result/) - `MatchResult` team breakdown + reward chips + leaderboard CTA via `ResultsPage`), a generalized bucketed **leaderboard** ([`instance/leaderboard/`](src/main/java/com/ziggfreed/common/instance/leaderboard/) - `Leaderboard` + `LeaderboardPage`, lifted from Kweebec), an asset-driven **reward** model with the no-claim-with-full-inventory guard ([`instance/reward/`](src/main/java/com/ziggfreed/common/instance/reward/) - `InstanceReward`/`InstanceRewardGranter`/`PendingRewardStore`, `InventoryUtil.canFit`), a codec-asset-driven **preset** ([`instance/preset/`](src/main/java/com/ziggfreed/common/instance/preset/) - `InstancePresetAsset` Pattern A + `InstancePresetConfig`), and the **generic Pattern-A registrar lift** ([`asset/`](src/main/java/com/ziggfreed/common/asset/) - `AssetStoreRegistrar`/`AssetMergeAdapter`). Each page takes an immutable `*PageDeps` + a locale-free `*Messages`/`*ScreenMessages` provider (the consumer supplies pre-built client-resolved `Message`s); pages reuse the shared `ZigFrames.ui`/`ZigButtons.ui` styles AND the lifted `ui/toast/` engine for in-page reward/feedback toasts (results reward strip, party invite/privacy actions, queue-joined). Reward chips render an item icon by id + an auto-generated "x{n} {ItemName}" label from the item's own engine display name (`RewardChipRenderer`; no per-reward `displayKey` needed). The party screen carries an owner public/private queue toggle. Kweebec's consumer wiring is its `experience/` package - the model for any future consumer.

> **The reusable ENCOUNTER / difficulty framework (for minigames, dungeons, raids, eventually the MMO).** Beyond the end-game layer, a consumer gets mod-agnostic primitives to BUILD an encounter: per-entity timed effects + escalation bands ([`instance/effect/`](src/main/java/com/ziggfreed/common/instance/effect/CLAUDE.md) - `EntityEffectService.applyTimed`/`applyBand` + `EffectBand`/`EffectBandLadder`, the on-hit slow / debuff / pace-band seam over `EffectControllerComponent`, lifting Kweebec's hunter-pace swap); weighted spawn rosters + a wave director ([`instance/encounter/`](src/main/java/com/ziggfreed/common/instance/encounter/CLAUDE.md) - `SpawnRoster<T>`/`SpawnUnit<T>` weighted + tier-gated picks, `EncounterDirector` cooldown/cap/per-round-max gating; PURE logic, no engine deps); floor-snapped runtime placement ([`world/SpawnPlacement`](src/main/java/com/ziggfreed/common/world/CLAUDE.md) - ring/near-player + foliage-skip surface snap over `SurfaceProbe`); a damage-cause index cache ([`util/DamageCauseCache`](src/main/java/com/ziggfreed/common/util/CLAUDE.md)); and a `RoundMetadata` outbound-event envelope ([`instance/metadata/`](src/main/java/com/ziggfreed/common/instance/metadata/CLAUDE.md)). **Kweebec Nightmare 0.5.0 is the exemplar consumer** (hunter on-hit slow + bonus damage + proximity-stacking + enrage; asset-driven spawn triggers near players on shrine-light + at corruption milestones; a multi-phase reskinned boss) - the SAME primitives back any future MMO dungeon/raid. Design + verified engine seams + in-game spikes: [[kweebec-difficulty-spawn-design]].

It has **zero MMO dependency and NO Perfect Utils coupling** - the only dependency is the Hytale server jar. The plugin entry registers nothing; every primitive is a static utility a consumer calls directly.

> **PARADIGM (remember this): a generic, mod-agnostic Hytale primitive belongs HERE, not duplicated in a consumer mod.** When a sibling mod (Kweebec Nightmare, a future minigame, eventually the MMO) needs a reusable engine wrapper - 3D sound, camera, an asset-index resolve, a HUD/toast helper, a worldgen/terrain query, etc. - add it to `ziggfreed-common` and consume it; do NOT reimplement it per mod. New primitive = a new file here (+ its package router), then rebuild the jar consumers pin. A consumer keeping only the mod-SPECIFIC policy (round logic, content config) is correct; the reusable engine call underneath is what gets lifted. (Latest example: `world/SurfaceProbe`, lifted out of Kweebec's `ArenaBuilder` so any mod can floor-snap onto procedural terrain.)

> **PARADIGM (the goal is no-Java pack authoring): consumer CONTENT + display CONFIG belongs in `/Server/` pack JSON, NEVER Java-baked.** Anything a pack author should be able to create or tune without code - dialogues, the queue-mode cards, leaderboard tabs/columns, party knobs, reward lists, queue policy - is a codec asset loaded from `Server/<Mod>/<Type>/` (Pattern A structured codec, or Pattern B raw-`Payload` + the dialogue template DSL), resolved `defaults < pack < owner`. The library ships at most a NEUTRAL structural fallback (e.g. `QueueModeSet.fallback()` has no baked icons) and ZERO content; the real values are authored in `/Server/ZiggfreedCommon/<Type>/`. A Java constant list / hardcoded JSON string that represents content is a smell - lift it to an asset. **Common OWNS its framework stores** (one class = one registrant; a consumer must NOT re-register them - it authors JSON into the path and reads the resolved config singleton back). The registered framework asset types ([`asset/FrameworkAssetRegistrar`](src/main/java/com/ziggfreed/common/asset/FrameworkAssetRegistrar.java), all via the [`asset/AbstractKeyedAssetConfig`](src/main/java/com/ziggfreed/common/asset/AbstractKeyedAssetConfig.java) fold + the Pattern-B [`asset/AbstractRawJsonAsset`](src/main/java/com/ziggfreed/common/asset/AbstractRawJsonAsset.java) base): **Dialogues**/**DialogueTemplates** ([`dialogue/asset/`](src/main/java/com/ziggfreed/common/dialogue/asset/), Pattern B + per-consumer-engine `resolveAll`), **Instances** ([`instance/preset/InstancePresetAsset`](src/main/java/com/ziggfreed/common/instance/preset/InstancePresetAsset.java)), **Bosses** (`instance/encounter/MultiPhaseBossAsset`), **BandedEffects** (`instance/effect/BandedEffectAsset` -> `EffectBandLadder`), **EncounterRules** (`instance/encounter/EncounterRuleAsset`, free-string Trigger), **PrefabPlacements** (`world/WeightedPrefabPlacementAsset`), **Leaderboard** (`instance/leaderboard/LeaderboardLayoutAsset`), **Party** (`party/PartySettingsAsset`). A consumer reading any of these must do so LAZILY / post-load (the configs are populated by `LoadedAssetsEvent` AFTER plugin `setup()`). Plan + remaining consumption work: [[ziggfreed-common-asset-driven-configs]].

## Build

Gradle runs via PowerShell (Java 25). Self-contained `build.ps1` builds + installs:
```powershell
cd 'D:\dev\business\hyMMO\additional-mods\ziggfreed-common'; .\build.ps1
.\build.ps1 -Install:$false     # build only
.\build.ps1 -ModsDir <path>     # explicit install target (else $env:HYTALE_MODS_DIR)
```
Produces `build/libs/ZiggfreedCommon-<version>.jar` and copies the runtime jar (never `-sources`/`-javadoc`) into the Hytale `Mods/` folder when `HYTALE_MODS_DIR` is set. `.\gradlew.bat build` works too. The Hytale server jar is referenced `compileOnly` (path in `gradle.properties`: `hytaleHome`/`patchline`/`game_build`) and NEVER bundled; jsr305 is `implementation` (the `@Nonnull`/`@Nullable` annotations must resolve for consumers that compile against this jar).

## Consuming this mod

A consumer mod adds ziggfreed-common as a `compileOnly files(...)` dependency (the local jar path until a remote + submodule gitlink exist) and lists it in its `manifest.json` `Dependencies` (or `OptionalDependencies`) so the server loads it first. NEVER bundle this jar into a consumer (double-loading engine-touching classes under two classloaders breaks identity). All primitives are static, config-free, and reusable as-is.

## Architecture (per-package routers carry the working detail)

Each domain package carries a nested `CLAUDE.md` router that loads when you touch that subtree. One line each:

- **[`sound/`](src/main/java/com/ziggfreed/common/sound/CLAUDE.md)** - `Sound3D`: play a `SoundEvent` by id at a position OR an entity ref, with a `SoundCategory` + an optional per-listener `Predicate<Ref>`; index resolution through `AssetIndexCache`. World-thread.
- **[`camera/`](src/main/java/com/ziggfreed/common/camera/CLAUDE.md)** - `CameraShakeService` (one-shot `CameraEffect` shake) + `ServerCameraService` (`SetServerCamera` top-down apply/reset). Packet-only; thread-safe writes.
- **[`util/`](src/main/java/com/ziggfreed/common/util/CLAUDE.md)** - `AssetIndexCache` (the "cache ONLY a positive index" resolver), `NumberFormatter`, `CommandExecutor` (execute-only), `HostilityUtil`, `EntityIdentifierUtil`, `DamageCauseCache` (memoized `DamageCause` index, for identifying a custom damage cause in a `DamageEventSystem`).
- **[`inventory/`](src/main/java/com/ziggfreed/common/inventory/CLAUDE.md)** - `InventoryUtil` (give/count/take/spend a custom resource item by id across the combined inventory sections; world-thread, try-guarded).
- **[`feedback/`](src/main/java/com/ziggfreed/common/feedback/CLAUDE.md)** - `Notify` (Default/Danger/Warning/Success toasts) + `EventTitles` (centered banner). Both take a pre-built `Message`.
- **[`ui/`](src/main/java/com/ziggfreed/common/ui/CustomHudHelper.java)** - `CustomHudHelper` (install a consumer-built `CustomUIHud` + strip/restore the native HUD), plus **[`ui/toast/`](src/main/java/com/ziggfreed/common/ui/toast/CLAUDE.md)** - the lifted transport-agnostic in-page toast engine (`ToastController`/`ToastRenderer`/`ToastSpec`/`ToastLine`/`ToastKind` + a `ToastablePage<T>` page base, shipping its own neutral `Pages/ZigToast.ui`). i18n is parameterized OUT (pre-built `Message`s only). The hyMMO MMO is the shared-file owner and re-derives its own `ToastablePage` over this engine.
- **[`world/`](src/main/java/com/ziggfreed/common/world/CLAUDE.md)** - `SurfaceProbe` (top-solid-Y column probe via `World.getBlock`, skipping transparent foliage; world-thread, try-guarded) to floor-snap runtime-placed prefabs/entities onto procedural terrain; `SpawnPlacement` (ring-around / near-player runtime spawn positions, each surface-snapped with a foliage-skip set so a runtime spawn lands on GROUND, not canopy).
- **[`instance/effect/`](src/main/java/com/ziggfreed/common/instance/effect/CLAUDE.md) + [`encounter/`](src/main/java/com/ziggfreed/common/instance/encounter/CLAUDE.md) + [`metadata/`](src/main/java/com/ziggfreed/common/instance/metadata/CLAUDE.md)** - the reusable ENCOUNTER framework (see the blockquote above): `EntityEffectService`/`EffectBand`/`EffectBandLadder` (timed + banded per-entity effects over `EffectControllerComponent`), `SpawnRoster<T>`/`SpawnUnit<T>`/`EncounterDirector` (weighted tier-gated spawn picks + cooldown/cap/per-round-max wave gating, pure logic), `RoundMetadata` (outbound integration envelope). Kweebec 0.5.0 is the exemplar; the MMO's dungeons/raids are the target.
- **[`worldmap/`](src/main/java/com/ziggfreed/common/worldmap/CLAUDE.md)** - `WorldMapMarkers`: place/remove world-map + compass POI markers over the native `WorldMapManager` - global (all players, via the engine `POIMarkerProvider`) AND per-player (`registerProvider` returning each player's own markers). Build a marker with `marker(id, icon, x,y,z, name)`. Needs the world's compass/map enabled to render.
- **[`dialogue/`](src/main/java/com/ziggfreed/common/dialogue/CLAUDE.md)** - the generic branching NPC dialogue engine: a per-consumer `DialogueEngine` (builder pre-seeds generic `Goto`/`Close`/`SetFlag`/`Talk`/`OpenPage` actions + `Flag`/`NotFlag` `Type`-list conditions, the consumer registers domain types + seams), the executor/sugar/template-DSL/structural-validator, and a `page/DialoguePage` whose `.ui` ships in this jar. The hyMMO MMO consumes + extends it; a minigame builds with generics only. NOT a static-util like the others - it is built per consumer.

## Conventions

`@Nonnull`/`@Nullable` on params; `ZiggfreedCommonPlugin.LOGGER` for logging (guard the raw LOGGER behind a try/catch on any path a unit test could reach - the flogger LOGGER throws in a log-manager-less unit JVM). `ConcurrentHashMap` for shared maps. Every engine-touching call is try-guarded so a missing asset / bad ref degrades to a no-op, never a throw into the caller. World-thread discipline: off-thread work hops via `world.execute` before any Store/Ref/Sound/HUD/camera read; packet writes (`writeNoCache`/`write`) are thread-safe off-thread. **No em-dashes anywhere.** Package root `com.ziggfreed.common`. **UI RICH TEXT (any primitive that shows text): colour/bold/italic = native `<color is="#hex">`/`<b>`/`<i>`/`\n` markup in the `.lang`, rendered on a Label's `TextSpans` (NEVER `.Text`, which prints the tags literally); per-locale, client-parsed; `TextButton`s + String sinks can't. The `dialogue/` page does this (`#NodeText.TextSpans`). Crucial UX infra, authority: `../../.claude/research/hytale-rich-text-textspans.md`.**

**Submodule order (when a remote exists):** commit + push HERE first, verify the SHA is on the remote, THEN bump the gitlink in the parent hyMMO repo. Until a remote exists, consumers build against the local jar.
